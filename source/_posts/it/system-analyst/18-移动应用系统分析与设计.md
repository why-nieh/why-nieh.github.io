# 18.1 移动应用平台概述
- **移动端应用开发平台**：
  - EMAS（阿里巴巴，全端）
  - TMF（腾讯，一站式）
  - FinClip（小程序容器）
  - APICloud（跨平台引擎）
  - EMOP（京东，移动中台）
- **移动端应用部署平台**：
  - 小程序部署平台：微信小程序 vs 支付宝小程序（设计逻辑、面向人群、功能区）
  - App部署平台：Android vs iOS（开源/闭源、硬件效率、运行机制、后台、屏幕响应、开放性）

# 18.2 移动应用开发环境
- **App开发环境**：
  - Android：JDK、JRE、JVM、Eclipse/IDEA、Android Studio
  - iOS：Xcode、iOS SDK、Objective-C/Swift
- **小程序开发环境**：
  - 微信开发者工具：WXML、WXSS、JS、JSON，模拟器
- **网页应用开发环境**：HTML5开发工具、移动设备设计技术

# 18.3 移动应用架构
- **发展历史**：
  - 单体应用
  - MVC架构（Model-View-Controller）
  - MVP架构（Model-View-Presenter）
  - MVVM架构（Model-View-ViewModel）
  - 微服务架构
- **案例**：
  - 微信（MVC）
  - 美食家（MVP）
  - 知乎iOS（MVVM + RxSwift）
  - Netflix（微服务）
  - VUE（MVVM）
  - 滴滴出行（MVP + 敏捷开发）

# 18.4 移动应用开发
- **开发语言**：
  - 原生：Java/Kotlin（Android）、Swift/OC（iOS）
  - WebApp：HTML/CSS/JS
  - H5：纯网页
  - 混合/混编：HTML/JS + Native.js
  - 桥接App：JS编译为原生（React Native）
  - Flutter：Dart语言，自绘引擎
- **小程序开发方式**：
  - 基于微信框架（MINA）
  - 原生开发（SDK）
  - 网页开发（嵌入WebView）
  - 跨平台（React Native、Weex）
  - 云开发（云函数、数据库）
- **原生App开发**：
  - Android：Java/Kotlin，需求、交互设计、UI设计、开发环境
  - iOS：Objective-C/Swift，Xcode、Cocoa Touch、硬件接口、CoreData、WebKit
  - 案例：滴滴出行（MVP + 微服务）、知乎iOS（MVVM）
- **网页App开发**：
  - WebApp定义：基于浏览器的应用
  - 案例：去哪儿网（React Native）
- **混合App开发**：
  - 优点：跨平台、一次开发多处运行、可调用原生API
  - 缺点：性能不如原生、依赖WebView
  - 框架：Ionic、React Native、Flutter
  - 案例：Uber（Ionic）

# 18.5 无代码开发
- **发展阶段**：4个阶段（代码生成工具→业务开发平台→模型驱动→无代码云平台）
- **优劣**：
  - 优点：降低门槛、提高效率、减少成本
  - 缺点：灵活性有限、依赖第三方、可能成本高
- **主流平台**：
  - 开发平台：Airtable、Bubble、Microsoft PowerApps、Google App Maker
  - 测试平台：TestComplete、Ranorex Studio
- **开发流程**：需求分析、界面设计、逻辑实现、数据管理、测试调优、发布维护
- **案例**：Zapier（触发器、动作、映射、命名激活、管理、订阅、开放平台）
