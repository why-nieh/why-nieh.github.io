# 9.1 信息系统安全体系
- **安全威胁**：系统破坏、设备滥用、信息泄露等
- **系统安全分类**：
    - 实体安全：环境、设备、媒体
    - 运行安全：风险管理、审计跟踪、备份恢复、应急
    - 信息安全：操作系统安全、数据库安全、网络安全、病毒防护、访问控制、加密、认证
    - 人员安全：意识、法律、技能
- **安全体系结构**：
    - 物理层安全
    - 操作系统层安全
    - 网络层安全
    - 应用层安全
    - 安全管理
- **安全保护等级**（GB17859—1999）：
    1. 用户自主保护级
    2. 系统审计保护级
    3. 安全标记保护级
    4. 结构化保护级
    5. 访问验证保护级
- **安全保障系统**：
    - 统一身份认证
    - 统一安全管理
    - 规范的安全保密体系
    - 完善的网络边界防护

# 9.2 数据安全与保密
- **数据加密技术**：
    - 传统加密：代换、置换
    - 对称加密：DES、IDEA（分组密码）、RC4（序列密码）
    - 非对称加密：RSA（基于大数分解）、ECC（椭圆曲线）
- **认证技术**：
    - **数字签名**：功能（鉴别、不可否认、完整性）
    - **杂凑算法**：MD5、SHA家族（SHA-1、SHA-2）
    - **数字证书**：X.509格式，CA签发，CRL列表
    - **身份认证方式**：
        - 口令认证
        - 动态口令（OTP）
        - 生物特征识别（指纹、虹膜、人脸）
- **密钥管理体制**：
    - **KMI**：密钥分发中心（KDC），静态/动态分发
    - **PKI**：公钥基础设施，CA、证书库、密钥备份、作废系统、应用接口
    - **SPK**：种子化公钥（LPK、CPK）

# 9.3 通信与网络安全技术
- **防火墙**：
    - 功能：访问控制、内容过滤、日志、NAT、VPN
    - 分类：包过滤、电路级网关、应用网关、代理、状态检测、自适应代理
    - 体系结构：双宿/多宿主机、屏蔽主机、屏蔽子网（DMZ）
    - 局限性：不能防内部攻击、不能防病毒、影响性能
- **虚拟专用网（VPN）**：
    - 关键技术：隧道、加解密、密钥管理、身份认证、访问控制
    - PPP会话阶段：创建链路、用户验证、PPP回叫、调用网络层协议
- **安全协议**：
    - **SSL**：握手协议、记录协议、警报协议；三层服务（认证、加密、完整性）
    - **HTTPS**：HTTP over SSL，端口443
    - **PGP**：邮件加密，RSA + 对称加密
    - **IPSec**：IP层安全，认证头（AH）、封装安全载荷（ESP）

# 9.4 单点登录技术
- **SSO概念**：一次登录，访问多个应用
- **Kerberos**：基于对称加密的第三方认证，票据（Ticket）
- **外壳脚本机制**：通过脚本激活目标平台账号

# 9.5 系统访问控制技术
- **访问控制要素**：主体、客体、控制策略
- **访问控制策略**：
    - 登录访问控制
    - 操作权限控制
    - 目录安全控制
    - 属性安全控制
    - 服务器安全控制
- **访问控制模型**：
    - **BLP模型**：保密性，上读、下写
    - **Lattice模型**：安全类别，基于格
    - **Biba模型**：完整性，下读、上写
- **访问控制分类**：
    - **DAC**：自主访问控制（文件权限）
    - **MAC**：强制访问控制（多级安全）
    - **RBAC**：基于角色的访问控制（角色、权限、会话）
    - **TBAC**：基于任务的访问控制（动态）
    - **OBAC**：基于对象的访问控制（ACL与对象关联）

# 9.6 容灾与业务持续
- **灾难恢复技术**：
    - **RPO**：恢复点目标，数据丢失量
    - **RTO**：恢复时间目标，业务停顿时间
    - **灾难恢复等级**（GB/T20988—2007）：6个等级
    - **容灾分类**：数据容灾、应用容灾
- **灾难恢复规划**：
    - 确定需求（风险分析、业务影响评估、确定目标）
    - 制定策略（7个要素）
    - 实现策略（技术方案、备份中心、技术支持、运维、预案）
    - 预案制定（起草、评审、测试、修订、审核批准）
- **业务持续性规划（BCP）**：
    - 阶段：项目启动、风险评估、业务影响分析、策略、开发BCP、培训、演练、持续维护
    - 主体框架：DRP、BRP、CMP、CA

# 9.7 安全管理措施
- **安全管理内容**：
    - 密码管理
    - 网络管理（配置、性能、安全、计费、故障）
    - 设备管理（选型、检测、安装、维护）
    - 人员管理（思想、职业道德、技能）
- **安全审计**：
    - 基本要素：控制目标、安全漏洞、控制措施、控制测试
    - CC标准6部分：自动响应、数据生成、分析、浏览、事件选择、事件存储

# 9.8 系统可靠性
- **系统故障模型**：
    - 故障分类：永久、间歇、瞬时
    - 故障等级：逻辑级、数据结构级、软件级、系统级
- **可靠性指标**：
    - MTTF（平均无故障时间）、MTTR（平均故障修复时间）、MTBF（平均故障间隔时间）
    - 可用性 = MTTF/(MTTF+MTTR)
- **可靠性模型**：
    - 时间模型：\(E_r(\tau) = E_0 / (I - E_c(\tau))\)
    - 故障植入模型：\(\hat{N}_0 = [m \times (n-k)/k]\)（植入错误法）；\(\hat{N}_0 = [m \times n / k]\)（两步查错法）
    - 数据模型：\(P_1 = |E_e|/|E|\)，\(R(n) = (1-P_1)^n\)
- **可靠性分析**：
    - 串联系统：\(R = R_1 \times R_2 \times ...\)
    - 并联系统：\(R = 1 - (1-R_1)(1-R_2)...\)
    - 模冗余系统：\(R = \sum_{i=n+1}^{m} C_m^i R_0^i (1-R_0)^{m-i}\)

# 9.9 冗余技术
- **冗余分类**：
    - 结构冗余：静态（三模）、动态（备用）、混合
    - 信息冗余：检错码、纠错码
    - 时间冗余：重复执行（复执）
    - 冗余附加：为实现冗余所需的资源
- **冗余系统故障处理10步骤**：
    1. 故障检测
    2. 故障屏蔽
    3. 故障限制
    4. 复执
    5. 故障诊断
    6. 系统重配置
    7. 系统恢复
    8. 系统重新启动
    9. 修复
    10. 系统重组合

# 9.10 软件容错技术
- **N版本程序设计**：
    - 不同小组独立设计，相异性规范评审、相异性确认、背对背测试
    - 需注意同步、通信、表决算法、一致比较问题、数据相异性
- **恢复块方法**：
    - 主块 + 后备块，验证测试决定是否切换
    - 与N版本比较：恢复块后向恢复，单机；N版本前向恢复，多机
- **防卫式程序设计**：
    - 错误检测（状态断言）
    - 破坏估计
    - 错误恢复（前向/后向）
