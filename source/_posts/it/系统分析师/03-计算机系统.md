#### 3.1 计算机系统概述
- **特点**：
  - 处理机管理（进程管理）
  - 存储器管理
  - 设备管理
  - 文件管理
  - 用户接口
- **层次结构**：
  - 硬件层（裸机）：硬联逻辑、微程序、传统机器
  - 系统层（系统软件）：操作系统、语言处理程序(编译程序、汇编程序、解释程序)
  - 应用层（应用软件）：应用程序
- **冯·诺依曼体系结构**：
  - 五大部件：运算器、控制器、存储器、输入设备、输出设备
  - 存储程序原理：程序和数据存放在同一存储器中。
- **计算机系统软件**：
  - 系统软件：用于实现计算机系统的管理、调度、监视和服务等功能。5类：操作系统、语言处理程序、服务性程序、数据库管理系统和计算机网络软件。
  - 应用软件：为解决某种应用问题而编制的一些程序。
  - 固件：存储在能永久保存信息的器件中的程序，是具有软件功能的硬件。优点：执行速度快于软件，灵活性优于硬件。
#### 3.2 存储器系统
- **三级**：高速缓冲存储器（Cache）、主存储器、辅存储器
- **存取**：4种：顺序存取、直接存取、随机存取和相联存取。
- **主存储器（Main Memory）**：
  - RAM：DRAM（需要刷新）、SRAM（不需刷新，速度快）
  - ROM：掩膜ROM、PROM、EPROM、EEPROM、Flash
- **辅助存储器**：
  - **磁带**：顺序存取，容量大，速度慢。
  - **硬盘**：
    - 直接存取 
    - 结构：盘片、磁头、主轴、控制器
    - 参数：柱面、磁道、扇区
    - 容量公式：\(存储容量 = n\times t\times s\times b\)
    - 存取时间：寻道时间 + 旋转等待时间 + 传输时间
    - 分类：HDD（机械）、SSD（固态）、SSHD（混合）
    - **机械硬盘**
      - \(存储容量 = 总记录面数n \times 每面磁道数t \times 每道扇区数s \times 每扇区字节数b\)
      - \(平均存取时间 T_a = 平均寻道T_s + 平均等待T_w = \frac{Tsmin+Tsmax}{2} + \frac{Twmin+Twmax}{2}\)
  - **磁盘阵列RAID**：
    - 独立磁盘冗余阵列，用多个较小的磁盘驱动器替换单一大容量磁盘驱动器，合理分布存放数据，支持同时
    - 主要技术：分块技术、交叉技术和重聚技术
    - 8个级别：
      - RAID0：条带化，无冗余，性能高
      - RAID1：镜像，安全性高，利用率50%
      - RAID2：海明码，用于早期，现少用，大量数据性能较高
      - RAID3：位交叉奇偶校验，独立校验盘，读数据快，写数据较慢（计算校验位）
      - RAID4：块交叉奇偶校验，独立校验盘，读数据快，写数据较慢（计算校验位）
      - RAID5：分布式奇偶校验，无独立校验盘
      - RAID6：双重奇偶校验，可容忍两个盘故障
      - RAID7：异步，高性能
      - RAID10：RAID1+0，高可靠+高性能
  - **光盘**：
    - CD-ROM、CD-R、CD-RW、DVD-ROM（单面/双面，单层/双层）
- **Cache**：
  - **局部性原理**：时间局部性、空间局部性
  - **命中率** \(h\)，平均访问时间 \(t_3 = Cache周期h\cdot t_1 + (1-h)\cdot 内存周期t_2\)
  - **映射方式**：
    - 直接映射：\(K = I \mod C\)（块号 = 主存页号 mod Cache块数）
    - 全相联映射：主存页可映射到任何Cache块
    - 组相联映射：\(J = I \mod Q\)（组号 = 主存页号 mod Cache组数），组内全相联
  - **替换算法**：随机、FIFO、LRU、NUR（最近未用）
  - **写策略**：
    - 写直达：同时写主存
    - 写回：仅写Cache，淘汰时写回主存
    - 标记法：设有效位，写时置无效
- **网络存储**：
  - **DAS**：直接附加存储，SCSI连接
  - **NAS**：网络附加存储，文件级共享（NFS、CIFS）
  - **SAN**：存储区域网络，块级访问
    - FC SAN：光纤通道
    - IP SAN：iSCSI
    - IB SAN：InfiniBand
- **虚拟存储**：
  - 概念：将多个物理存储抽象为统一存储池
  - 实现方式：
    - 主机级：卷管理软件
    - 设备级：存储控制器
    - 网络级：SAN中的虚拟化设备
  - 特点：集中管理、提高带宽、兼容性、资源利用率

#### 3.3 输入输出系统
- **I/O工作方式**：
  - **程序控制方式**：
    - 无条件传送：外设总是准备好
    - 程序查询方式：CPU查询外设状态，效率低
  - **程序中断方式**：
    - 中断过程：请求、判优、响应、处理、返回
    - 多中断处理：多中断线、中断软件查询、维菊链、总线仲裁、中断向量表
  - **DMA方式**：
    - CPU放弃对总线的控制，DMAC控制总线，直接内存访问
    - 获取总线方式：CPU停止访问主存法（暂停方式）、存储器分时法（共享方式）、周期挪用法（周期窃取方式）
    - 特点：速度快，CPU和外设并行
  - **通道方式**：
    - 高级I/O控制部件，更多的免去CPU的介入
    - 字节多路通道：低速设备共享
    - 选择通道：独占高速设备
    - 数组多路通道：分时共享，成组传送
  - **I/O处理机**：
    - 外围处理机，均有丰富的指令系统和完善的中断系统，基本上独立于中央处理机异步工作
    - 独立处理机，完成I/O任务
- **总线**：
  - 一组能为多个部件**分时**共享的公共信息传送线路
  - **分类**：
    - 功能：地址总线、数据总线、控制总线
    - 位置：内部总线、外部总线
    - 功用：局部总线、系统总线、通信总线
    - 数据线数量：并行总线、穿行总线
  - **标准**：ISA、PCI、AGP、PCIe等
  - **性能指标**：
    - 总线宽度：数据线位数
    - 总线带宽 = 宽度 × 频率（同步）
    - 总线负载：可连接设备数
    - 分时复用：同一线路不同时间传送不同信号
    - 猝发传输：一次地址后连续传送多个数据
- **I/O接口**：
  - **功能**：
    - 通信联络控制
    - 地址译码
    - 数据缓冲
    - 数据格式变换（并串转换、数模转换）
    - 传递控制命令和状态
  - **分类**：
    - 信号：数字、模拟
    - 通用性：通用、专用
    - 灵活性：可编程、不可编程
    - 数据交换方式：串行、并行
  - **常见接口**：
    - IDE/EIDE：硬盘接口
    - ATA：并行ATA，SATA串行
    - SCSI：小型计算机系统接口
    - PCMCIA：笔记本扩展接口
    - IEEE-1394：火线，高速串行
    - USB：通用串行总线，热插拔
  - **端口**：
    - 接口电路中CPU可以直接访问的寄存器，由若干个端口加上相应的控制逻辑电路醉成的接口
    - 分类：数据端口、命令端口、状态端口
    -编制：
      - 独立编址：I/O指令访问
      - 统一编址：内存映射，用访存指令

#### 3.4 指令系统
- 指令是指示计算机执行某些操作的命令，一台计算机的所有指令的集合构成指令系统（指令集）
- 直接运行的目标程序，由若干机器指令组成
- **基本指令系统**：
  - **设计要求**：完整性、规整性、高效率、兼容性
  - **指令类型**：
    - 数据传送类：一般传送、堆栈操作、数据交换
    - 运算类：算术、逻辑、移位（算数、逻辑、循环）
    - 程序控制类：转移、调用、循环
    - I/O指令
    - 处理机控制和调试指令（管态（特权指令）、用户态（一般指令））
- **CISC复杂指令系统**：
  - **特点**：
    - 指令数量多（100—250）
    - 使用频率相差悬殊，20%指令使用频率占80%
    - 寻址方式多（5—20）
    - 变长指令
    - 可对主存数据直接处理
    - 微程序控制为主
  - **缺陷**：
    - 80-20规律（使用频率悬殊）
    - 不利于VLSI（超大规模）
    - 增加硬件复杂度、执行指令周期长
- **RISC精简指令计系统**：
  - **特点**：
    - 指令数量少
    - 寻址方式少
    - 指令长度固定，格式种类少
    - 硬布线控制
    - 单周期执行，易利用流水线技术
    - 优化编译器
    - 通用寄存器多（>32）
      - 两个Cache：指令Cache和数据Cache
  - **核心思想**：减少CPI，\(P = I \times CPI \times T\)
    - RISC的I比CISC多20-40%，但CPI小（1左右），T小（RISC的T是CISC的1/3左右），综合速度提高3-5倍。
  - **关键技术**：
    - 延迟转移技术：在转移指令后插入有效指令
    - 指令取消技术：条件转移时决定后续指令是否取消
    - 重叠寄存器窗口：每个过程使用相邻窗口，减少参数传递开销
    - 指令流调整：编译器优化，减少数据相关
    - 硬件为主，固件为辅

#### 3.5 多处理机系统
- **多处理机与并行处理机比较**：
  - 结构灵活性：多处理机更灵活
  - 程序并行性：多处理机是任务级并行
  - 并行任务派生：多处理机需要专门指令
  - 进程同步：多处理机异步，需要同步机制
  - 资源分配：多处理机动态调度
- **分类**：
  - **共享存储（紧耦合）**：UMA、NUMA、COMA
    - UMA：均匀存储访问，所有处理器访问内存时间相同
    - NUMA：非均匀，访问本地内存快
    - COMA：只用高速缓存，组成全局地址空间
  - **分布式存储（松耦合）**：MPP（海量并行处理）
- **互连网络**：
  - **互连函数**：
    - 恒等：\(I(x)=x\)
    - 交换：\(E(x_1x_0)=x_1\overline{x_0}\)
    - 方体：\(C_k(x_{n-1}...x_k...x_0)=x_{n-1}...\overline{x_k}...x_0\)
    - 均匀洗牌：\(S(x_{n-1}x_{n-2}...x_0)=x_{n-2}...x_0x_{n-1}\)
    - 蝶式：\(B(x_{n-1}...x_0)=x_0x_{n-2}...x_1x_{n-1}\)
    - 位序颠倒：\(P(x_{n-1}...x_0)=x_0...x_{n-1}\)
  - **互连方式**：
    - 总线
    - 交叉开关
    - 开关枢纽
    - 多端口存储器
    - 多级互连网络

#### 3.6 操作系统
- **概述**：
  - **定义**：管理软硬件资源，提供用户接口
  - **特征**：并发、共享、虚拟、不确定性
  - **功能**：
    - 处理机管理：复杂对处理机分配和运行实施有效的管理
    - 存储器管理：对内存进行分配、保护和扩充
    - 设备管理：
      - 设备分配：按一定的原则对设备进行分配，缓冲技术、虚拟技术
      - 设备传输控制：实现物理的输入/输出操作，如：启动设备、 中断处理、结束处理等
      - 设备独立性：用户向系统申请的设备与实际操作设备无关
    - 文件管理：对文件存储空间进行管理
    - 用户接口：操作系统提供的一组友好的使用其功能的手段。包括程序接口和操作接口
- **分类**：
  - 批处理操作系统
  - 分时操作系统（多路性、独立性、交互性、及时性）
  - 实时操作系统（硬实时、软实时）
  - 网络操作系统（集中、C/S、对等）
  - 分布式操作系统（透明性、可靠性、高性能）
  - 嵌入式操作系统（微型化、可定制、实时性、可靠性、易移植）
- **进程管理**：
  - **前驱图**：有向无循环图，表示任务依赖关系
    - 程序并发执行的特征
      - 失去程序的封闭性
      - 程序和机器程序需的活动不再一一对应
      - 并发程序间的相互制约性
  - **进程组成**：
    - PCB：Process Control Block 进程控制块，进程存在的唯一标志
    - 程序：描述进程需要完成的功能，程序执行时不可修改的部分
    - 数据：包括程序执行时所需要的数据及工作区，每个进程专用，可修改部分
  - **状态模型**：
    - 三态：运行、就绪、阻塞
    - 五态：新建、就绪、运行、阻塞、终止
    - 新建→就绪→运行→终止
    - 运行→就绪：时间片到
    - 运行→阻塞→就绪→运行：运行过程遇需等待事件，变阻塞，等待事件发生，变就绪，等时间片，变运行
  - **信号量与PV操作**：
    - 整型信号量 \(S\)
    - P操作：\(S=S-1\)；若 \(S<0\) 则阻塞
    - V操作：\(S=S+1\)；若 \(S\le0\) 则唤醒一个等待进程
    - 互斥：\(P(mutex)\) 临界区 \(V(mutex)\)
    - 同步：生产者-消费者问题，信号量 \(S_1\)（空缓冲区数），\(S_2\)（满缓冲区数），\(mutex\)（互斥）
    - 缺点：编程难度大，效率低
  - **高级通信**：
    - 共享存储
    - 消息传递（Send、Receive通信原语）
    - 管道通信（pipe文件）
  - **死锁**：
    - 必要条件：互斥、不剥夺、请求与保持、环路
    - 鸵鸟测量：不予理睬策略
    - 预防：限制并发进程对资源的请求，破坏任一条件
      - 预先静态分配：所需资源有时不明确，降低资源利用率，降低并发程度
      - 资源有序分配：限制进程对资源的请求，因资源排序占用系统开销
    - 避免：设法破坏死锁条件，严格防止死锁的发生
      -银行家算法（安全状态、安全序列）
    - 检测与解除：允许死锁，定期检测，通过资源剥夺或撤销进程，解除死锁
  - **线程**：
    - 传统进程：可拥有资源的独立单位；可独立调度和分配的基本单位
    - 引入原因：减少进程切换开销
    - 进程作为独立分配资源的单位；线程作为调度和分配的基本单位。线程基本不拥有资源，同一进程的多个线程可共享其全部资源
    - 分类：用户级线程、内核支持线程
- **存储器管理**：
  - **分区管理**：固定分区、可变分区、可重定位分区（碎片问题）
  - **分页管理**：
    - 逻辑地址：页号 + 页内地址
    - 页表：页号 → 物理块号
    - 地址变换机构：页表寄存器（起始地址+长度），页号越界检查，得到物理块号，拼接页内地址
  - **分段管理**：
    - 逻辑地址：段号 + 段内地址
    - 段表：段号 → 段基址+段长
    - 地址变换：段号越界检查，段内地址越界检查，基址+段内地址
  - **段页式管理**：
    - 逻辑地址：段号 + 段内页号 + 页内地址
    - 段表：页表起始地址+页表长度
    - 地址变换：查段表得页表，查页表得物理块号，拼接页内地址
  - **虚拟存储**：
    - 局部性原理：时间局部性、空间局部性
    - 工作集：在某段时间间隔内进程实际访问的页面集合
    - 实现方式：请求分页、请求分段、请求段页式
    - 页面置换算法：
      - OPT：最佳置换，淘汰未来最远使用的页
      - FIFO：先进先出，有Belady异常
      - LRU：最近最少使用，需要硬件支持（寄存器或栈）
      - NUR：最近未用，访问位循环检查
- **设备管理**：
  - I/0系统：负责管理设备和输入/输出的机构
  - 组成：
    - 设备
    - 控制器
    - 通道
    - 总线
    - I/0软件
  - **I/O软件层次**：
    - 用户级软件
    - 与设备无关的系统软件
    - 设备驱动程序
    - 中断处理程序
    - 硬件
  - **SPOOLing技术**：
    - 组成：预输入程序、缓输出程序、井管理程序、输入井、输出井
    - 作业状态：提交、后备、执行、完成
  - **磁盘调度算法**：
    - FCFS：先来先服务
    - SSTF：最短寻道时间优先
    - SCAN（电梯算法）：单向移动，避免饥饿
    - C-SCAN：磁头单向扫描
- **文件管理**：
  - 实现文件统一管理的一组软件和相关数据的集合，专负责管理和存取文件信息的软件机构
  - **功能**：
    - 按名存取
    - 统一的用户接口
    - 并发访问和控制
    - 安全性控制
    - 性能优化
    - 差错恢复
  - **文件结构**：
    - **逻辑结构**：流式文件、记录式文件
    - **物理结构**：
      - 连续结构
      - 链接结构（隐式链接、显式链接）
      - 索引结构（单级索引、多级索引）
      - UNIX三级索引：直接索引、一级间接、二级间接、三级间接
  - **文件目录**：
    - FCB：文件控制块
    - 目录结构：一级、二级、多级（树形）
    - 路径名：绝对路径、相对路径
  - **存储空间管理**：
    - 位示图
    - 空闲区表
    - 空闲块链
    - 成组链接法（UNIX）
  - **文件共享**：
    - 硬链接（基于索引节点）
    - 符号链接（软链接）
  - **文件保护**：
    - 存取控制矩阵：用户、文件、权限三维矩阵
    - 存取控制表：按权限对用户进行分类
    - 用户权限表：按用户（组）
    - 密码
  - **安全**：
    - 安全性管理：系统级、用户级、目录级、文件级
  - **可靠性**：
    - 系统抵抗和预防各种物理性破坏和人为性破坏的能力
    - 保护信息：转储、恢复、日志文件和文件系统的一致性检查（块一致性检查和文件一致性检查）
- **作业与用户界面**：
  - 作业：系统为完成一个用户的计算任务（或一次事务处理）所做的工作的综合。
    - 由程序、数据和作业说明书组成
  - **作业控制**：
    - 脱机控制：使用JCL编写作业说明书，连同作业一并提交
    - 联机控制：操作系统提供用户一组联机命令
  - **作业状态**：提交、后备（收容）、执行、完成
  - **作业调度算法**：
    - 因素：与系统的整个设计目标一致；均衡地使用系统资源；平衡系统和用户的要求
    - FCFS
    - 短作业优先
    - 优先级
    - 响应比高优先 \(R_p = 1 + \frac{\text{等待时间}}{\text{执行时间}}\)
  - **用户界面**：计算机中实现用户与计算机通信的软/硬件部分的总称
    - 命令接口、程序接口（系统调用）、图形界面
    - 控制面板式用户界面→字符用户界面→图形用户界面→新一代用户界面（VR）
