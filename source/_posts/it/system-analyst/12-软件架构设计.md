# 12.1 软件架构概述
- **意义**：交流手段、早期设计决策、约束实现、决定开发组织、制约质量属性、简化更改、支持原型、培训基础、可复用模型
- **发展史**：无架构→萌芽（程序结构）→初级阶段（UML）→高级阶段（4+1视图）

# 12.2 软件架构建模
- **模型类型**：结构模型、框架模型、动态模型、过程模型、功能模型
- **“4+1”视图模型**：
  - 逻辑视图（功能需求，类图）
  - 开发视图（模块组织，子系统图）
  - 进程视图（运行特性，并发、分布）
  - 物理视图（部署，节点、网络）
  - 场景（用例，将四个视图联系起来）

# 12.3 软件架构风格
- **定义**：惯用模式，构件+连接件+约束
- **数据流风格**：
  - 批处理：顺序执行，整体传递
  - 管道-过滤器：过滤器（处理）、管道（传输）
- **调用/返回风格**：
  - 主程序/子程序
  - 面向对象：对象封装数据+操作
  - 层次型：上层调用下层
  - C/S：二层、三层、B/S
- **以数据为中心风格**：
  - 仓库：中央数据 + 独立构件
  - 黑板：多个知识源共享黑板
- **虚拟机风格**：
  - 解释器：解释引擎、存储区、状态、进度
  - 规则系统：规则集、解释器、选择器、工作内存
- **独立构件风格**：
  - 进程通信：消息传递
  - 事件系统：隐式调用，事件注册、触发
- **面向服务的架构（SOA）**：
  - 服务基本结构：接口、实现
  - 设计原则：明确定义接口、自包含、粗粒度、松耦合、互操作性
  - 关键技术：UDDI、WSDL、SOAP、REST
  - 实现方法：WebService、服务注册表、企业服务总线（ESB）

# 12.4 软件架构标准
- IEEE 1471-2000/ISO/IEC/IEEE 42010：
  - 任务、环境、系统、架构
  - 利益相关者、架构说明、原理
  - 关注、关注点、视图
  - 关注点库、模型

# 12.5 软件架构实现
- **ABSDM模型**：6个子过程（需求、设计、文档化、复审、实现、演化）
- **体系结构需求**：
  - 需求获取（质量目标、商业目标）
  - 标识构件（生成类图、分组、打包）
  - 需求评审
- **体系结构设计**：
  - 提出模型（选择风格）
  - 映射构件
  - 分析相互作用
  - 产生体系结构
  - 设计评审
- **体系结构文档化**：规格说明、质量设计说明书
- **体系结构复审**：外部人员参与，识别风险
- **体系结构实现**：
  - 基于文档化的体系结构
  - 查找或开发构件
  - 组装
  - 测试
- **体系结构演化**：
  1. 需求变化归类
  2. 制订演化计划
  3. 修改、增加、删除构件
  4. 更新构件相互作用
  5. 构件组装与测试
  6. 技术评审

# 12.6 软件架构质量属性
- **质量属性概念**：
  - 开发期：易理解、可扩展、可重用、可测试、可维护、可移植
  - 运行期：性能、安全性、可伸缩性、互操作性、可靠性、可用性、健壮性
- **面向评估的质量属性**：
  - 性能、可靠性、可用性、安全性、可修改性、功能性、可变性、互操作性
- **质量属性场景描述**：
  - 刺激源、刺激、环境、制品、响应、响应度量
  - 6类场景：可用性、可修改性、性能、可测试性、易用性、安全性（表格）
