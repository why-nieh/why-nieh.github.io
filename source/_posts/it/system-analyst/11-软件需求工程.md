# 11.1 软件需求概述
- **需求层次**：业务需求、用户需求、系统需求
- **质量功能部署（QFD）**：常规需求、期望需求、意外需求
- **软件需求特性**：正确、无歧义、可检验、可跟踪、完整、一致、可修改、可理解
- **需求工程**：获取、分析、规格说明、验证、管理

# 11.2 需求获取
- **用户访谈**：准备、主持、后续
- **问卷调查**：设计问题、类型（开放式/封闭式）
- **采样**：简单随机、分层、聚类、系统
- **情节串联板**：被动、主动、交互式
- **联合需求计划（JRP）**：JAD会议，6~18人，1~5小时，步骤（互相认识、开放讨论、设想、整理）
- **需求记录技术**：
  - 任务卡片（目的、触发、频率、子任务、变体）
  - 场景说明（用户场景描述）
  - 用户故事（6属性：独立、可协商、有价值、可预测、短小、可测试）
  - Volere白卡（规范模板）

# 11.3 需求分析
- **任务**：整理、分析、形成软件需求规格说明
- **分析方法**：
  - SA（结构化分析）
  - OOA（面向对象分析）
  - PDOA（面向问题域分析，问题框架）

# 11.4 结构化分析方法
- **SA模型**：数据字典为核心，数据模型（E-R）、功能模型（DFD）、行为模型（STD）
- **数据流图（DFD）**：见10.6.4
- **状态转换图（STD）**：
  - 状态、事件、转换、初始状态、最终状态
  - 描述对象生命周期
- **数据字典**：
  - 条目：数据元素、数据结构、数据流、数据存储、加工逻辑、外部实体
  - 作用：列表、相互参照、检索、一致性检验
  - 管理：专人维护

# 11.5 面向对象分析方法
- **用例模型**：
  - 参与者：人、其他系统、硬件、时钟
  - 用例命名：动词+名词
  - 用例规约：名称、简要说明、事件流（主/备）、非功能需求、前置/后置条件、扩展点、优先级
  - 用例关系：包含、扩展、泛化
- **分析模型**：
  - 定义概念类：名词短语法，筛选
  - 确定类之间的关系：关联、依赖、泛化、聚合、组合、实现
    - 关联细节：名称、角色、多重性、导向性
  - 为类添加职责：属性、方法
  - 建立交互图（顺序图、通信图）

# 11.6 需求定义
- **定义方法**：
  - 严格定义（假设所有需求可预先定义、交流准确、图形充分体现）
  - 原型方法（原型演示、迭代改进）
- **软件需求规格说明书（SRS）**：
  - 编写方法：文本、图形、形式化
  - 内容和格式（GB/T8567—2006）：范围、引用文件、需求、合格性规定、可追踪性、未解决问题、注解、附录

# 11.7 需求验证
- **需求评审**：
  - 技术评审类型：评审、检查、走查
  - 做好评审的经验：分层次、正式与非正式结合、分阶段、精心挑选人员、培训、检查单、标准流程、跟踪、充分准备
- **需求测试**：
  - 概念测试用例：基于用例，描述测试脚本
  - 需求测试过程：测试人员根据概念用例执行，发现遗漏、错误，修改需求

# 11.8 需求管理
- **需求变更**：
  - 变更控制过程：问题分析、变更分析、变更实现
  - 变更控制委员会（CCB）：成员、决策过程、交流
  - 变更策略：所有变更必须遵循过程、未批准不实现、CCB决定、风险承担者了解、不删除原始文档、可追踪
- **需求跟踪**：
  - 正向跟踪：需求→设计→代码→测试
  - 逆向跟踪：工作成果→需求
  - 需求跟踪矩阵
