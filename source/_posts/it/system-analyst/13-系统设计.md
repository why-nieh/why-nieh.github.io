# 13.1 系统设计概述
- **概要设计**：系统总体结构设计，模块划分、模块调用关系
- **详细设计**：具体技术手段（网络、代码、输入/输出、处理流程、数据存储、用户界面、安全可靠性）

# 13.2 处理流程设计
- **流程概念**：
  - 流程定义、流程实例
  - 工作流：部分或完全自动化的业务过程
  - 活动、所有者、工作项
- **工作流管理系统（WFMS）**：
  - 基本功能：建模、执行、管理和分析
  - 组成：工作流执行服务、工作流引擎、流程定义工具、客户端应用、调用应用、管理监控工具
  - 5个接口：定义交换、客户端应用、调用应用、互操作、系统管理
- **流程设计工具**：
  - 程序流程图（5种基本结构：顺序、IF-THEN-ELSE、DO-WHILE、DO-UNTIL、DO-CASE）
  - IPO图
  - N-S图（盒图）
  - PAD（问题分析图）
  - PDL（过程设计语言）
  - 判定表、判定树

# 13.3 结构化设计
- **模块结构**：
  - 信息隐蔽与抽象
  - 模块化原则：大小适中、扇入扇出合理、深度宽度适当
  - 耦合（7种，从低到高）：非直接、数据、标记、控制、外部、公共、内容
  - 内聚（7种，从高到低）：功能、顺序、通信、过程、瞬时、逻辑、偶然
  - 模块类型：传入、传出、变换、协调
- **系统结构图（SC）**：
  - 组成：模块、调用关系、通信、辅助控制（菱形表示条件，弧形表示循环）
  - 变换型：输入、变换、输出三部分
  - 事务型：事务中心、事务层、操作层、细节层
  - 混合型：先变换后事务或反之

# 13.4 面向对象设计
- **设计软件类**：
  - 实体类（持久化）
  - 控制类（业务逻辑）
  - 边界类（界面、接口）
- **对象持久化与数据库**：
  - ORM（对象关系映射）
  - 持久化框架：CMP、Hibernate、iBatis、JDO
- **OOD原则**：
  1. 开闭原则（对扩展开放，对修改关闭）
  2. 里氏替换原则（子类可替换父类）
  3. 依赖倒置原则（依赖抽象，不依赖具体）
  4. 组合/聚合复用原则（优先组合，少用继承）
  5. 接口隔离原则（多个专用接口）
  6. 最少知识原则（迪米特法则）

# 13.5 设计模式
- **概述**：模式名称、问题、解决方案、效果
- **分类**：
  - 创建型：工厂方法、抽象工厂、原型、单例、建造者
  - 结构型：适配器、桥接、组合、装饰、外观、享元、代理
  - 行为型：职责链、命令、解释器、迭代器、中介者、备忘录、观察者、状态、策略、模板方法、访问者

# 13.6 输入/输出原型设计
- **输入设计原则**：
  - 确定数据类型和格式
  - 确定数据来源
  - 设计良好界面（布局、控件、提示、错误提示）
  - 设计输入验证机制
  - 设计输入处理流程
  - 设计输入存储方案
  - 设计输入安全机制
- **输出设计原则**：
  - 简洁性
  - 一致性
  - 可用性
  - 可访问性
  - 可维护性
  - 安全性
  - 可扩展性

# 13.7 人机交互设计
- **设计目标**：满足需求、易用、降低错误率
- **用户体验五层模型**：
  1. 战略层：用户需求、产品目标
  2. 范围层：功能、内容
  3. 结构层：交互设计、信息架构
  4. 框架层：界面、导航、信息设计
  5. 表现层：视觉设计
- **如何设计良好的人机交互**：
  - 了解用户（细分、研究、人物角色）
  - 设计符合习惯的界面
  - 提供即时反馈
  - 辅助功能（为残障人士）
  - 用户测试
  - 迭代优化
  - 多语言支持
  - 结合实际应用场景
  - 适当使用动画
  - 参考新理念、新技术（AI、VR）
