# 6.1 企业信息化概述
- **定义**：利用信息技术控制和管理企业信息，实现内外部信息共享和有效利用。
- **常用方法**：
  - 业务流程重组：对企业组织结构和工作方法进行“彻底的、根本性的”重新设计，以适应当今市场发展和信息社会的需求
  - 核心业务应用：围绕核心业务应用信息技术
  - 信息系统建设
  - 主题数据库：面向企业额业务主题的数据库，也是面向企业核心业务的数据库
  - 资源管理：信息技术的应用为企业资源管理提供了强大的支持

# 6.2 信息资源管理
- **IRM（Information Resource Managerment）内容**：
  - 数据资源管理：数据控制（**维护**和**安全**）
  - 信息处理管理：管理人员获取和处理信息（**流程**和**方法**），并且强调切中信息资源的重要性
  - 三个主题：
    - 资源管理的方向和控制，从整个企业管理的层面来分析资源的管理。数据可共享、数据处理机构提出应用项目、资源的有效性
    - 建立企业信息资源知道委员会，负责制定方针政策，控制和监督信息资源功能的实施
    - 建立信息资源的组织机构，从事数据的计划和控制、数据获取以及数据的经营管理，并包括企业应用系统的开发
  - 起点和基础：建立信息资源目录
  - 关键环节：制定IRM标准

|数据库管理|数据管理|
|----|----|
|仅仅负责物理数据库的设计、实现、安全性和维护工作|确定数据规划、数据应用、数据标准、数据内容、数据范围等|
- **信息资源分类**：
  - 管理维度
    - 信息资源都是业务信息化基础上形成的各种信息。分类：一是专门的业务部门所采用的分类体系；二是综合部门从信息资源登记和管理的角度提出的分类
  - 来源维度
    - 按照信息资源提供部门来设置信息资源的一级、二级乃至三级类目
  - 应用主题维度
- **IRM基础标准**：
  - 数据元素标准（命名、标识、一致性）
  - 信息分类编码标准（ABC分类法）
  - 用户视图标准（表单、报表、屏幕）
  - 概念数据库标准（主题数据库概要）
  - 逻辑数据库标准（基本表规范）
- **制定和实施原则**（威廉·德雷尔10条）：
  1. 不能把例外当成正规
  2. 管理部门支持
  3. 标准切实可行
  4. 灵活余地
  5. 不迁就落后
  6. 方法越多越有效
  7. 宣传推广，不强迫
  8. 细节不重要，重要的是有标准
  9. 逐渐制定，渐进执行
  10. 一致性最重要

# 6.3 信息资源规划（IRP，Information Resource Planning）
- 是信息化建设的基础工程，是指对企业声场经营活动所需的信息，在产生、获取、处理、存储、传输和利用等方面进行全面的规划
- **关键步骤**：
  1. 定义职能域
  2. 各职能域业务分析
  3. 各职能域数据分析
  4. 建立企业级IRM基础标准
  5. 建立信息系统功能模型
  6. 建立信息系统数据模型
  7. 建立关联矩阵
- **业务主线**：职能域-业务过程-业务活动
  - 职能域模型
  - 业务过程
  - 业务活动
  - 业务模型复查
- **数据主线**：
  - 用户视图采集、整理、分析（数据需求分析）
  - 建立基本表和主题数据库（系统数据建模）

# 6.4 信息资源网建设
- 信息通信网：利用国家公用电信平台（公用电话网、公用分组交换网和数字数据网等）构筑的区域性或专业性的通信网。将信息以某种形式进行传输
- 信息资源网：指各相关经济信息提供部门和使用部门建立的各种数据库、信息中心和信息应用系统。包括从信息采集和加工，直至最终使用的环节
- 数据库和数据仓库的建设
- 数据分布策略：分散存储、集中存储
- 分布式数据库建设

# 6.5 信息化规划
- 信息化规划是企业信息化建设的纲领和向导，是信息化系统设计和实施的前提和依据。是以整个企业的发展目标和战略、企业各部门的目标与功能为基础，同时结合行业信息化方面的实践和对信息技术发展趋势的掌握，制定出企业信息化远景、目标和发展战略，从而达到全面、系统地指导企业信息化建设的目的，充分而有效的利用企业的信息资源，已全面满足企业发展的需求
- **信息化规划内容**：
  - 业务流程重组
    - 理解关键的企业目标
    - 企业战略规划：企业如何到达目标。在符合和保证现实企业使命的条件下，在充分利用环境中存在的各种机会和创造新机会的基础上，确定企业与环境的关系，确定企业从事的业务范围、成长方向和竞争对策，合理地调整企业结构和分配企业的全部资源
    - 信息系统战略规划：信息系统如何支撑这些目标
    - 信息技术战略规划（IT战略规划）：需要哪些信息技术支撑信息系统
      - 业务分析
      - 检查当前IT架构和系统
      - 识别计划
      - 选择方案
  - 信息化资源规划
    - 信息资源规划：信息化建设具体项目的实施
  - 在进行信息化规划时，需：明确发展目标和实施重点、成立领导机构、做好企业业务信息化需求分析、确定企业信息化不同发展阶段的投资预算和制定必要的促进企业信息化建设的规章制度

- **信息化战略与企业战略关系**：保证企业信息化战略和企业战略的一致性，把企业战略的目标转换为信息系统的战略目标。一方面，随着信息技术的迅猛发展和普及，企业战略的实现已经离不开信息技术和信息系统；另一方面，离开企业战略规划区规划信息系统，犹如建设海市蜃楼
- **信息化规划与企业战略规划**：互相影响，相互促进
  - 信息化初级阶段：各部门根据业务流程或管理需要，提出信息化需求，分别独立实施
  - 信息化中级阶段：企业制定了整体战略规划，业务部门根据企业战略，优化业务流程、组织结构，独立实施
  - 信息化高级阶段：企业根据整体战略规划，通盘考虑个业务部门信息化需求，制定整体的信息化战略，统一规划，分步实施
- **信息化战略与企业战略的集成**：
  - 信息化战略从企业战略出发，服务于企业战略，同时又影响和促进企业战略。
  - 集成方法
    - **业务与IT整合**（BITA，Business-IT Alignment）：
      - 一种以业务为导向的、全面的IT管理咨询实施方法论。从制定企业战略、建立（或改进）企业组织结构和业务流程，到进行IT管理和制定过渡计划，是IT能够更好的为企业战略或目标服务
      - 适用：信息系统不能满足当前管理中的业务需要，业务和IT之间总是有不一致的地方的企业
      - 步骤：评估和分析企业当前业务和IT不一致的领域，整理出企业的业务远景和未来战略，建立业务模型，提出达到未来目标的转变过程建议和初步计划，以及执行计划
    - **企业IT架构**（EITA，Enterprise IT Architecture）
      - 分析企业战略，帮助企业制定IT战略，并对其投资决策进行指导。在技术、信息系统、信息、IT组织和IT流程方面，帮助企业建立IT的原则规范、模式和标准，支出IT需要改进的方面并帮助制定行动计划
      - 适用：现有信息系统和IT基础架构不一致、不兼容和缺乏统一的整体管理的企业
- **信息系统战略规划方法（ISSP，Information System Strategic Planning）**：
  - 从企业战略出发，构建企业基本的信息架构，对企业内、外信息资源进行统一规划、管理与应用，利用信息控制企业行为，辅助企业进行决策，帮助企业实现战略目标
  - 第一阶段：以数据处理为核心，围绕只能部门需求进行的信息系统规划
    - **BSP（企业系统规划，Business System Planning）**：目标是提供一个信息系统规划，用以支持企业短期的和长期的信息需求
      - 原则：信息系统必须支持企业的战略目标；信息系统的战略应表达出企业各个管理层次（战略计划层、管理控制层、操作控制层）的需求；信息系统应该向整个企业提供一致的信息；信息系统应该适应企业组织结构和管理体制的改变；信息系统战略规划应当有总体信息系统结构中的子系统开始实现（自上而下规划，自下而上实现）
      - 步骤：准备工作、定义企业过程、定义数据类、分析现有系统、确定管理部门要求、提出判断和结论、定义信息总体结构、确定优先顺序、评价信息资源管理、制订建议书和开发计划、成果报告。
      - 产出：CU矩阵（过程/数据类表）、PO矩阵（过程/组织）、RD矩阵（资源/数据）。
    - **CSF（关键成功因素法， Critical Success Factors）**：
      - CSF确定（行业、企业、部门、管理人员）
      - 步骤：了解企业目标、识别CSF、识别性能指标、定义数据字典。
      - 缺点：分析过程主观、个人倾向。
    - **SST（战略集合转化法， Strategy Set Transformation）**：
      - 步骤：识别企业战略集合（使命、目标、战略），转化为信息系统战略集合（目标、环境约束、战略规划）。
      - 与BSP、CSF比较：CSF突出重点，SST全面，BSP核心是过程。
  - 第二阶段：主要是以企业内部管理信息系统为核心，围绕企业整体需求进行的信息系统规划
    - **SDP（战略数据规划法, Strategy Data Planning）**：
      - 企业模型建立（职能范围、业务过程、业务活动）
      - 主题数据库（面向业务主题，共享，一次一处输入，基本表组成）
      - 数据环境分类：文件、应用数据库、主题数据库、信息检索系统
      - 执行过程：实体分析、实体活动分析、企业重组、亲和度分析、分布数据规划
    - **IE（信息工程法, Information Engineering）**：
      - 组成：系统的方法论、完备的工具集、信息工程环境和成熟的经验总结
      - 企业信息系统三要素：信息（各种信息）、过程（企业的业务活动过程）和技术（企业采用的信息技术） 
      - 实施阶段：信息战略规划、业务领域分析、系统设计、系统构建
        - 步骤：信息战略规划、业务领域分析、业务系统设计、技术系统设计、系统构建、系统转换和系统运转
      - 信息战略规划报告：
        - 摘要
          - 信息战略规划所涉及的范围
          - 企业的业务目标和战略重点
          - 信息技术对企业业务的影响
          - 对现有信息环境的评价
          - 推荐的系统战略（关于信息结构规划和业务系统结构规划的总结）
          - 推荐的技术战略（关于技术结构的总结）
          - 推荐的组织战略（关于企业进行机构改革的建议）
          - 推荐的行动计划（要执行的主要项目、项目的持续时间、硬件设备获得的时间）
        - 规划
          - 阐述总体内容，业务环境描述；
          - 评价现有信息环境，确定满足业务环境需要方面存在的问题；
          - 通过可选择方案和推荐的信息结构、业务系统结构、技术结构，阐明其优点，确定问题的解决方案；
          - 给出推荐的行动计划
        - 附录
    - **SG（战略栅格法, Strategic Grid）**：
      - 四象限：
        - 支持型：现有信息系统对战略影响程度低，规划中的信息系统对战略影响程度低
        - 工厂型：现有信息系统对战略影响程度高，规划中的信息系统对战略影响程度低
        - 转变型：现有信息系统对战略影响程度低，规划中的信息系统对战略影响程度高
        - 战略型：现有信息系统对战略影响程度高，规划中的信息系统对战略影响程度高
      - 根据象限确定规划方法
  - 第三阶段：在综合考虑企业内、外环境的情况下，以集成为核心，围绕企业战略需求进行信息系统规划
    - **VCA（价值链分析法， Value Chain Analysis）**：
      - 识别增值环节（制造、采购等）
      - 识别减值环节
    - **SAM（战略一致性模型/战略对应模型， Strategy Alignment Model）**：
      - 外部领域：
        - 企业经营战略：企业对产品和市场在竞争领域的定位选择问题，包括企业目标、核心能力和管理制度三个方面
        - 信息系统战略：企业在IT市场中的定位选择，包括企业对信息系统目标、信息系统能力和信息系统管理制度方面的选择
      - 内部领域：
        - 组织与业务流程：企业的内部资源，他对企业所选择的市场竞争战略提供有效的支持，体现资源整合战略观，包括企业基础设施、业务流程和组织技能三个方面
        - IT基础架构：企业进行信息化建设的基础，包括IT基础设施、IT流程和IT技能三个方面
      - 一致性：战略匹配、功能整合
  - 规划方法的选择
    - 保证业务流程与信息系统架构之间的一致: BSP、SDP
    - 保证企业战略与信息系统战略之间的一致：SST、SG、CSF、VCA
    - 保证企业战略、业务流程和信息系统架构三者之间的一致: 以IE方法为代表


# 6.6 企业资源规划
- 企业资源：支持企业业务活动和战略运营的事物，包括人、财、物和信息。三个“流”：物流、资金流和信息流
- **ERP（Enterprise Resource Planning）概念**：管理思想、软件产品、管理系统
- **ERP功能模块**：
  - 财会管理（会计核算、财务管理）
  - 物流管理（分销、库存、采购）
  - 生产控制管理（主生产计划、物料需求计划、能力需求计划、车间控制、制造标准）
  - 人力资源管理（招聘、工资、工时、差旅）
- **ERP开发方法**：
  - 二次开发（基于套装软件）：开发周期短、投资小；套装软件在系统设计采用企业先进管理理念，与企业原来的业务实践不一致，要求企业向这种标准靠拢，很难满足不同企业的个性化需求
  - 定制开发：能够满足特定企业的需求；很难全面考虑系统的扩展性、稳定性等架构因素，系统不能快速适应企业的需求变化，开发周期长，效率不高，投资较大，实施风险大
- **ERP实施方法**：
  - 明确观点、统一认识、建立实施团队
  - 明确目标和制定实施计划
  - 根据企业人员知识结构和技术水平组织培训
  - 根据企业现状进行业务需求分析
  - 根据需求分析结果建模和进行原型分析
  - 根据实际业务流程和具体情况进行系统功能和参数配置以及系统实施
  - 根据业务原型进行试运行试验
  - 制定技术解决方案
  - 调试环境，培训和测试
  - 上线准备，数据准备
  - 系统上线，投入运行
  - 系统优化，周期性系统运行审查

# 6.7 企业信息系统
- **OA（Office Automation，办公自动化）**
- **CRM（Customer Relationship Management，客户关系管理）**：
  - 功能：客户服务、市场营销、共享客户资料库、分析能力
  - 解决方案：触发中心（沟通渠道）、挖掘中心（数据分析）
  - 实施过程：客户服务与支持、客户群维系、商机管理
- **SCM（Supply Chain Management，供应链管理）**：
  - 内容：计划、采购、制造、配送、退货
  - 关键问题：配送网络重构、配送战略、供应链集成、库存控制、产品设计、信息技术、顾客价值
- **PDM（Product Data Management）**：产品数据管理，管理所有与产品相关的信息和过程
- **PLM（Product Lifecycle Management）**：
  - 主要功能：总体功能（数字化产品模型、过程自动化）、核心创造/协作/控制、细化功能（文档管理、搜索引擎、工作流管理）
- **知识管理**：
  - 知识分类：显性知识、隐性知识（技术层面、认知层面）
  - 知识管理工具：知识生成、知识编码、知识转移
- **商业智能（Business Intelligence，BI）**：
  - BI技术：数据仓库、OLAP、数据挖掘
  - 实施步骤：需求分析、数据仓库建模、数据抽取、建立分析报表、用户培训、系统改进
- **企业门户（Enterprise Portal，EP）**：
  - 分类：企业网站、企业信息门户（EIP，Enterprise Information Portal）、企业知识门户（EKP，Enterprise Knowledge Portal）、企业应用门户（EAP，Enterprise Application Portal）
  - 实施关键：单点登录、业务流程整合、个性化配置、与企业应用集成、知识转化
- **决策支持系统（Decision Support System，DSS）**：
  - 决策分类：结构化、半结构化、非结构化
  - DSS功能：管理信息、存储模型、灵活运用模型、数据通信
  - 基本结构：数据库子系统、模型库子系统、推理部分、用户接口
- **信息系统开发方法**：
  - 结构化方法（生命周期法）：SA Structured Analysis结构化分析，SD Structured Design 结构化设计，SP Structured Programming 结构化程序设计
  - 面向对象（Object-Oriented）方法：OOA Object-Oriented Analysis 面向对象的分析、OOD Object-Oriented Design 面向对象的设计、OOP Object-Oriented Programming 面向对象的程序设计
  - 面向服务方法（SO，Service-Oriented）
  - 原型化方法（快速原型）

# 6.8 电子商务（EC，Electronic Commerce）
- **类型**：
  - B2C：Business to Customer，借助于Internet开展在线销售活动
  - B2B：Business to Business，两个或若干个有业务联系的企业通过B2B模式彼此连接起来，形成网上的虚拟企业圈
  - C2C：Customer to Customer，C2C平台就是通过为买卖双方提供一个在线交易平台，使卖方可以主动提供商品在网上拍卖，而买方可以自行选择商品进行竞价
  - B2G：Business to Government
  - C2B：Customer to Government
- **标准**：国际标准（用户接口、基本功能、数据及客体的定义与编码）、国家电子商务标准体系
- **特征**：普遍性、方便性、整体性、安全性、协调性
- **优势**：降低营销成本、提高销售效率、扩大渠道、提高服务质量、大数据分析
- **电子商务技术**：网络通信、EDI、安全、电子支付、数据库

# 6.9 电子政务
- **业务模型**：政务信息查询、公共政务办公、政府办公自动化
- **应用模式**：
  - G2G（政府对政府，Government to Government）：政府上下级之间、不同地区和不同职能部门之间实现电子政务活动
  - G2B（政府对企业，Government to Business）：政府向企业提供各项公共服务，主要包括向政府向企事业单位发布的各种方针、政策、法规和行政规定
  - G2C（政府对公众，Government to Citizen）：政府面向公众所提供的服务，主要是信息服务，还包括户口、各种证件的管理等政府提供的各种服务
  - G2E（政府对公务员，Government to Employee）：内部效率效能电子政务模式，是G2G、G2B、G2C的基础，主要利用Internet建立起有效的行政办公和员工管理体系，提高政府工作效率和公务员管理水平
- **实施**：国家电子政务标准体系建设

# 6.10 业务流程重组（BPR，Business Process Reengineering）
- 针对切业务流程的基本问题进行反思，并对它进行彻底的重新设计，使业绩取得显著性的提高
- **BPR概念**：根本性、彻底性、显著性、流程
- **BPR原则**：以流程为中心、团队式管理、以顾客为导向
- **BPR实施**：方法（研究现有流程、从零开始构建）、失败原因（更多的是人为因素）
- **基于BPR的信息系统规划**：
  - 关系：相互作用，相辅相成。一方面，信息系统规划要以BPR为前提，并且在系统规划的整个过程中，以业务流程为主线。另一方面，面向流程的信息系统规划驱动企业的BPR。
  - 步骤：
    - 战略规划。主要是明确企业的战略目标，认清企业的发展方向，了解企业运营模式;进行业务流程调查，确定成功实施企业战略的因素，并在此基础上定义业务流程，制定信息系统战略规划，使得信息系统目标与企业目标保持一致，为业务流程实施提供战略指导。
    - 流程规划。业务流程规划是数据规划与功能规划的基础，主要任务是选择核心业务流程，并进行流程分析，识别出关键业务流程，以及需要改进的业务流程，画出改进后的业务流程图。
    - 数据规划。在业务流程规划的基础上，识别由流程所产生、控制和使用的数据，并对数据进行相应的分类。首先定义数据类，然后进行数据的规划，按时间长短可以将数据分为历史数据、年报数据、季报数据、月报数据、日报数据等；按数据是否共享可以分为共享数据和内部专用数据；按数据的用途可分为系统数据、基础数据和综合数据等。
    - 功能规划。在对数据类和业务流程了解的基础上，建立数据类与过程的CU矩阵，对它们的关系进行综合，并通过CU矩阵识别子系统，进一步进行系统总体逻辑结构规划，即功能规划，识别功能模块。
    - 实施规划。本阶段包括两个活动，分别是确定系统开发顺序和制订项目开发计划，在企业目前资源有限的状况下，要确定各个信息系统开发的优先次序，保证那些最关键的信息系统能优先开发。同时，要制订各个信息系统开发的计划，保证信息系统战略能有序地实施。

# 6.11 企业应用集成（EAI， Enterprise Application Integration）
- **传统EAI**：
  - 表示集成（界面集成）：把原有零散的系统建模集中在一个新的界面中，黑盒集成，无需了解程序与数据库的内部构造。
    -适用：在现有的基于终端的应用系统上配置基于PC的用户界面；为用户提供一个看上去统一，但由多个系统组成的应用系统；当只有可能在显示界面上实现集成时
  - 数据集成（共享数据）：对数据进行标识并编成目录，确定元数据模型，保证数据在数据库系统中分布和共享，白盒集成。是控制集成和业务流程集成的基础。
    - 适用：需要对多种信息源产生的数据进行综合分析和决策；要处理一些多个应用需要访问的公共信息库；当需要从某数据月获取数据来更新另一个数据源时，特别是它们之间数据格式不相同时。
  - 控制集成（功能集成）：也称功能集成或应用集成。是在业务逻辑上对应用系统进行集成。实现控制集成时，可借助远程过程调用或远程方法调用、面向消息的中间件、分布式对象技术和事务处理监控器来实现
    - 适用：表示集成和控制集成适用的其都适用，单复杂度更高
  - 业务流程集成（过程集成）：
  - 企业间集成
- **事件驱动的EAI（Even-Driven Architecture）**：
  - EDA（事件驱动架构）：异步、发布/订阅、解耦
  - 事件驱动的EAI框架：
    - 企业服务总线（ESB Enterprise Service Bus）：面向服务体系中的基础架构，各个服务通过总线来互相访问
    - 应用服务层：指需要集成的各个应用系统和数据库
    - 总线接入层（适配器）：提供适配器服务，支持多种主流应用的接入协议，是用户可以访问各个应用服务，并通过消息机制使各种应用程序接入ESB，使用ESB的各种服务
    - 核心服务层（消息分发、队列、目录、数据转换）：提供多种ESB所需的必要的服务支持
    - 业务支持层（业务流程、用户交互）：侧重在业务支持上，通过通用、标准的对象和服务模型，可以在这一层上定义可复用的和基于企业标准的业务流程；提供统一的用户交互服务
