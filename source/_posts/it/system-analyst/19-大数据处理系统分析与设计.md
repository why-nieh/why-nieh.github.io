# 19.1 大数据处理系统概述
- 定义：处理海量数据的软件工具，分布式计算，支持商业智能、金融、医疗等

# 19.2 大数据处理系统架构
- **架构原则**：
  - 可扩展：分布式、模块化
  - 可管理：自动化、统一平台、可视化
  - 数据安全：加密、访问控制、备份恢复、审计、监控
  - 高性能：数据分区、分布式计算、并行处理、内存计算
  - 高可用：数据冗余、自动故障转移、负载均衡
  - 稳定性：监控预警、性能调优、故障排除、升级维护
- **架构类型**：
  - 批处理架构（离线，MapReduce）
  - 流处理架构（实时，Flink、Storm）
  - 混合架构（Lambda、Kappa）
- **架构模式**：
  - **Lambda架构**：
    - 批处理层（预计算，Hive、Spark SQL）
    - 加速层（流处理，Storm、Spark Streaming）
    - 服务层（查询，Kylin、Presto）
  - **Kappa架构**：
    - 流处理层（处理所有数据）
    - 在线服务层（存储结果）
  - **IOTA架构**：
    - 通用数据模型
    - 边缘服务器
    - 实时数据缓存区
    - 历史数据
    - 数据导入组件
    - 查询引擎
    - 实时反馈

# 19.3 大数据处理系统开发
- **数据存储**：
  - 数据采集：离线采集（ETL）、实时采集（Kafka）、互联网采集（爬虫）
  - 数据清洗：5步骤（分析、定义规则、搜寻错误、纠正、回流）
  - 数据转换：6步骤（发现、映射、提取、代码生成、代码执行、审查）
  - 数据存储：行存储 vs 列存储（对比表）
- **数据管理**：
  - 元数据管理：技术元数据、业务元数据，7种管理方式
  - 数据关系图谱：血缘关系（归属性、多源性、可追溯性、层次性）
  - 数据安全：认证+授权，数据分级（S4~S1）
  - 资源监控：底层基础、服务状态、组件性能、Runtime、集群指标、任务状态、趋势预测
- **数据处理**：
  - 实时计算（Flume+Kafka+Storm/Spark）
  - 离线计算（Hadoop MapReduce、Hive）
  - 图计算（Spark GraphX、图算法）
- **数据分析**：
  - 机器学习（监督、非监督、强化）
  - 搜索推荐（ElasticSearch、Solr、Nutch）
  - 数据可视化（Silk、Tableau、Datawrapper、Chartio）
- **系统部署**：
  - 部署分析（需求、模型、硬件、软件）
  - 部署原则：生产/测试隔离、集群隔离、在线/离线隔离、数据隔离
  - 部署环境（Linux、集群规划、网络、安全、时间同步、SSH）
  - 版本控制（Git、DVC、Pachyderm）

# 19.4 大数据处理系统测试
- **测试特点**：分布式、多样化数据类型、实时/批处理、高性能可扩展、复杂逻辑
- **测试过程**：需求分析、计划、用例设计、执行、结果分析
- **功能测试**：
  - 数据提取测试（查询、过滤、分区、聚合、排序、多表关联）
  - 数据处理测试（清洗、转换、聚合、分析）
  - 数据存储测试（容量、读写、冗余、一致性、压缩）
  - 数据迁移测试（导出导入、格式、速度、一致性、监控、回滚）
- **性能测试**：
  - 负载测试
  - 压力测试
  - 并发性测试
  - 延迟测试
  - 吞吐量测试
  - 性能指标监测
  - 可扩展性测试
- **可靠性和容错性测试**：
  - 故障注入
  - 容错恢复
  - 故障切换
  - 数据一致性
  - 容量和负载
  - 自动化监控恢复
- **安全性测试**：
  - 访问控制
  - 数据保护（加密、脱敏、备份）
  - 漏洞扫描/渗透测试
  - 弱密码测试
  - 防火墙/网络安全
  - 安全日志和监控
  - 安全策略和合规性
  - 社会工程学
- **兼容性测试**：
  - 硬件
  - 操作系统
  - 数据库
  - 第三方组件
  - 文件格式
  - API
  - 平台
