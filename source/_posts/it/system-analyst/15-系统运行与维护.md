# 15.1 运维技术指标
- **MTTR**（平均故障修复时间）
- **MTBF**（平均故障间隔时间）
- **MTTF**（平均无故障时间）
- **MTTA**（平均应答时间）

# 15.2 系统运行管理
- **系统用户管理**：
  - 统一用户管理
  - 身份认证方法：用户名/密码、IC卡、动态密码、USB Key
  - 用户安全审计
- **网络资源管理**：
  - 性能管理、故障管理、配置管理、计费管理、安全管理
  - 网络资源管理系统
- **软件资源管理**：
  - 软件构件管理（构件库）
  - 软件分发管理（打包、分发、安装、配置）
  - 文档管理

# 15.3 系统故障管理
- **故障监视**：
  - 设置待监视项目
  - 监视内容：人员、操作、系统硬件、软件
- **故障调查**：
  - 收集信息（自动/人工）
  - 确定故障位置
  - 调查故障原因（7类）
- **故障排查和恢复处理**：
  - 硬件恢复（备用、冗余）
  - 数据库恢复（事务、系统、介质）
  - 应用软件恢复（调试、容错）
- **故障收尾**：确认解决，更新记录

# 15.4 软件系统维护
- **软件维护性**：6个子特性（模块化、可重用、易分析、易修改、易测试、维护性依从性）
- **可维护性度量**：
  - 外部：MTTR
  - 内部：环路数、软件规模
- **维护分类**：
  - 改正性维护
  - 适应性维护
  - 完善性维护
  - 预防性维护
- **维护影响因素**：业务因素、理解局限性、优先级、积极性、测试困难
- **提高可维护性措施**：采用软件工程、注重可维护性的开发过程
- **软件维护管理**：
  - 维护组织
  - 维护工作流程（用户申请→审核→计划→实施→测试→确认→更新文档）
  - 保存维护记录
  - 评价维护活动（度量指标）

# 15.5 系统评价
- **信息系统建设**：监理的必要性
- **信息系统评价**：
  - 评价流程（确定对象、方案、实施、复核）
  - 评价指标：
    - 系统性能（可靠性、效率、可维护性等）
    - 系统效益（经济效益、社会效益）
    - 系统建设（阶段评审）
  - 指标设计：效率驱动、能力驱动、效果驱动、效益驱动
  - 指标类型：静态/动态、定量/定性、原生/衍生等
  - 权重设计：主观赋权（专家调查、层次分析）、客观赋权（主成分、熵值）、组合赋权
- **系统改进建议**：评价报告，提出建议

# 15.6 遗留系统处置
- **遗留系统评价**：
  - 启动评价（问题）
  - 业务价值评价
  - 外部环境评价（硬件、支撑软件、IT基础设施）
  - 应用软件评价（系统级、部件级）
  - 分析评价结果（技术水平 vs 业务价值，分四类）
- **遗留系统演化**：
  - 淘汰策略
  - 继承策略
  - 改造策略
  - 集成策略

# 15.7 新旧系统转换
- **转换策略**：
  - 直接转换
  - 并行转换
  - 分段转换（按功能、部门、设备）
- **数据转换和迁移**：
  - 方法：工具迁移、手工录入、新系统生成
  - 准备工作：数据源说明、数据字典、差异分析、映射、ETL工具、测试计划、应急措施
  - 转换与迁移：抽取、清洗、转换、装载
  - 校验：质量分析、新旧系统对比

# 15.8 现有系统演进
- **系统扩展**：动态/静态扩展，延伸型/新建型扩展
- **扩展与集成的比较**：
  - 扩展：深度维护，重点在设计
  - 集成：广度维护，重点在分析
