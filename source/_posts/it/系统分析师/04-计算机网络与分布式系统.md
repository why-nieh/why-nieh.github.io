# 4.1 计算机网络基础
- **两层**：
  - 内层通信子网的节点计算机与高速通信线路组成独立的数据系统，承担全网的数据传输、交换、加工和变换等通信处理工作
  - 外层资源子网包括计算机、终端、通信子网接口设备、外部设备及各种软件资源等，负责全网的数据处理和向网络用户提供网络资源及网络服务
- **数据通信基础**：
  - 信源：通信中产生和发送信息的一端
  - 信宿：接收信息的一端
  - 信道：信源和信宿之间的通信线路
  - 信道带宽：模拟信道 $W=f_2-f_1$，数字信道：最大数据速率
  - **奈奎斯特定理**：最大码元速率 $B=2W(Baud)$
  - **数据速率**：$R = B\log_2 N = 2W\log_2 N$，其中 $N$ 是码元离散值的个数
  - **香农定理**：$C = W\log_2(1+\frac{S}{N})$，$\frac{S}{N}$ 为信噪比，$dB=10\log_{10}(\frac{S}{N})$
- **误码率**：$P_c = \frac{N_e(出错的位数)}{N(传送的总位数)}$
- **信道延迟**：传播时延 = 距离 / 传播速度（电缆约200m/μs，卫星270ms）

# 4.2 数据编码
- **模拟数据编码**：
  - ASK（幅移键控）：调幅，技术简单，抗干扰能力差
  - FSK（频移键控）：调频，技术简单，抗干扰能力较强
  - PSK（相移键控）：调相，抗干扰能力较强，实现技术复杂
- **数字数据编码**：
  - NRZ（不归零）：低电平0，高电平1；同步信号
  - ME曼彻斯特编码：每个比特中间有跳变，高→低为1，低→高为0
  - DME差分曼彻斯特编码：比特开始有跳变表示0，无跳变表示1；中间跳变只作时钟
  - 多电平编码：编码的码元可取多个电平之一，每个码元可代表几个二进制位。提高频带利用率，抗噪声特性不好
  - 4B/5B编码：5位码表示4位数据，保证传输同步

# 4.3 差错控制
- **噪声**：
  - 由热噪声引起的随机错误
    - 热噪声是由电子热运动产生，热噪声时刻存在，具有很宽的频谱，且幅度较小，随机性，个别位，持续存在，影响全局
  - 由冲击噪声引起的突发错误
    - 外界电磁干扰，冲击噪声持续时间短，幅度大，位串出错，突发性，影响局部
- **奇偶校验**：增加一位使1的个数为奇数（奇校验）或偶数（偶校验）
- **海明码**：`检错`  `纠错`
  - 海明距离：两个码字之间不同位的个数
  - `纠错`：要纠正 $t$ 位错误，需要距离 $d\ge 2t+1$
  - 编码：设数据位 $m$，校验位 $k$，满足 $m+k+1 \le 2^k$ 
  - 步骤：将位编号`1~n`，把编号写成二进制，每个2的幂次位作校验位，校验位覆盖其编号二进制中该位为1的数据位。
- **CRC**： `检错`
  - 生成多项式 $G(x)$
  - 发送数据多项式 $D(x)$，计算 $x^k D(x) / G(x)$ 得余数$R(x)$，发送$x^k D(x)+R(x)$
  - 接收端用 $G(x)$ 除接收数据，余数为0则无错
  - 常用生成多项式：CRC-16、CRC-32等

# 4.4 网络体系结构与协议
- **网络体系结构**：计算机网络的各层及其协议的集合。
- **网络协议**：用以规定信息的格式以及如何发送和接收信息的一套规则
  - 语法：控制信息或数据的结构和格式
  - 语义：需要发出何种控制信息、完成何种动作以及做出何种应答
  - 同步：时间实现顺序的详细说明
- **OSI参考模型**：
  - 物理层：比特流传输，机械、电气、功能、过程特性
  - 数据链路层：帧传输，差错控制、流量控制（HDLC、PPP）
  - 网络层：包传输，分组传输，路由选择、拥塞控制（IP），内连接，提供两个终端用户之间数据传输的通路
  - 传输层：端到端可靠传输（TCP、UDP）
  - 会话层：会话管理（建立、维护、同步）
  - 表示层：数据格式转换、加密、压缩
  - 应用层：网络应用协议（HTTP、FTP、SMTP）
- **TCP/IP协议簇**：
  - TCP/IP协议分为4层：应用层、传输层、网际层、网络接口层。不含硬件层
  - **应用层**：对应ISO模型的应用层、表示层和会话层，HTTP、FTP、SMTP、POP3、DNS、SNMP
  - **传输层**：对应用层传递的信息进行分段处理，并加入附加说明，TCP、UDP
    - **TCP**：面向连接，可靠，段格式（源端口、目的端口、序号、确认号、数据偏移、保留、标志URG/ACK/PSH/RST/SYN/FIN、窗口、校验和、紧急指针、选项）
    - **UDP**：无连接，不可靠，段格式（源端口、目的端口、长度、校验和）
  - **网际层**：将传输层形成的一段一段的信息，构造为IP数据报，在报头中填入地址信息，然后选择好发送的路径，IP、ICMP、ARP、RARP
    - **IP协议**：无连接，不可靠，数据报格式（版本、首部长度、服务类型、总长度、标识、标志、片偏移、生存时间、协议、首部校验和、源地址、目的地址、选项）
    - **ICMP**：差错报告、网络探测（ping）
    - **ARP**：IP地址→MAC地址
    - **RARP**：MAC地址→IP地址
  - **网络接口层（链路层）**：接收和发送IP数据报，负责与网络中的传输媒介打交道 

# 4.5 网络地址
- **IP地址分类**：
  - A类：0开头，8位网络号，24位主机号（1.0.0.0~127.255.255.255）
  - B类：10开头，16位网络号，16位主机号（128.0.0.0~191.255.255.255）
  - C类：110开头，24位网络号，8位主机号（192.0.0.0~223.255.255.255）
  - D类：1110开头，组播地址
  - E类：1111开头，保留
- **子网划分**：
  - 子网掩码：网络号和子网号部分全1，主机号部分全0
  - 子网地址 = IP地址 & 子网掩码
- **CIDR**：
  - 无分类编址，IP地址/网络前缀长度
  - 路由汇聚：将多个连续网络聚合成一条路由

# 4.6 局域网与广域网
- **局域网拓扑**：星形、总线、环形、网状
- **以太网**：
  - CSMA/CD：载波监听多路访问/冲突检测
    - 监听算法：非坚持、1-坚持、P-坚持
    - 冲突后退：二进制指数退避
  - 10Base5（粗缆）、10Base2（细缆）、10Base-T（双绞线）、10Base-F（光纤）
  - 快速以太网：100Base-TX、100Base-FX、100Base-T4
  - 千兆以太网：1000Base-SX（多模光纤）、1000Base-LX（单模光纤）、1000Base-CX（同轴）、1000Base-T（双绞线）
- **无线局域网**：
  - IEEE802.11（a/b/g/n/ac）
  - 拓扑：点对点、HUB型、完全分布型
- **广域网**：
  - SONET/SDH：同步光网络
  - DDN：数字数据网
  - 帧中继：X.25简化版，虚电路
  - ATM：异步传输模式，53字节信元

# 4.7 网络工程
- **网络规划**：
  - 需求分析：
    - 功能需求：用户希望网络完成什么功能
    - 通信需求：用户需要的通信类型、通信频度、通信时间和通信量等
    - 性能需求：容量（带宽）、利用率、吞吐量、可提供负载、效率、延迟（等待时间）、延时变化量、响应时间、端到端的差错率和网络效率
    - 可靠性需求：精确度、错误率、稳定性、无故障时间、数据备份等几个方面
    - 安全需求：可用性、完整性和保密性
    - 运行与维护需求：网络运行和维护费用方面
    - 管理需求：用户管理、资源管理、配置管理、性能管理、网络维护
  - 可行性研究：技术、经济、法律、用户使用
  - 现有网络分析：
    - 服务器的数量和位置
    - 客户端的数量和位置
    - 同时访问的数量：并发峰值
    - 每天的用户数
    - 每次使用的时间
    - 每次数量传输的数据量
    - 网络拥塞的时段
    - 采用的协议
    - 通信模式
    - 不要推到重来，要基于现有设备的基础上进行升级和改造。将现有的设备降级使用，并新增更加先进的设备，以提高网络的性能
- **网络设计**：
  - 在网络规划的基础上，设计一个能够解决用户问题的方案。
  - **网络设计的任务**
    - 确定网络总体目标：网络技术、标准，构筑规模，是否分期实施，实施成本，运行成本
    - 确定总体设计原则：设计原则的选择和权衡。实用性原则、开发性原则、高可用性/可靠性原则、安全性原则、先进性原则、易用性原则和可扩展性原则
    - 通信子网设计：拓扑结构与网络总体规划、分层的设计，远程接入访问的设计
    - 资源子网设计：服务器的接入和子网连接的问题
    - 设备选型：网络设备和各个层次的交换机的选择策略
    - 网络操作系统与服务器资源设备：根据具体网络应用，确定操作系统，再确定服务器
    - 网络安全设计：确定面临的攻击和风险，明确安全策略，建立安全模型，选择并实现安全服务，对安全产品的选型进行测试
  - **分层设计**：
    - 核心层（高速交换）：通过高速转发通信，提供优化、可靠的骨干传输结构。应拥有更高的可靠性、性能和吞吐量。
      - 双星（树）结构：采用两台相同的交换机，与汇聚层分别连接，并使用链路汇聚技术实现双机互联
    - 汇聚层（策略控制）：接入层与核心层的分界面，完成网络访问策略控制、数据包处理、过滤、寻址，以及其他数据处理的任务。应具备更高的性能、更少的接口和更高的交换速率
    - 接入层（用户接入）：直接面向用户或网络连接部分。目的是允许终端用户连接到网络。应具备低成本、高端口密度特性
  - 设备选型：交换机、路由器、防火墙
  - 安全设计：访问控制、VPN、防火墙策略
- **网络实施**：
  - 工程实施计划
  - 设备到货验收
  - 设备安装（布线、机房）
  - 系统测试（设备测试、系统测试、应用测试）
  - 系统试运行
  - 用户培训
  - 系统转换（直接、并行、分段）

# 4.8 分布式系统
  - 组件分布在互联网的计算机上，组件之间通过消息传递进行通信和动作协调的系统
  - 关键特征：组件的并发性（进程间并发）、缺乏全局时钟和组件故障的独立性等
  - 主要动力：对共享资源的期望
- **定义与特征**：
  - 异构性：不同实现，使用公共标准
  - 开放性：发布系统的关键接口；基于一致的通信机制和发布接口访问共享资源；可以基于异构硬件和软件构造系统
  - 安全性：机密性（防泄漏）、完整性（防止被改变或破坏）、可用性（防止对访问资源的干扰，如DDoS攻击）
  - 可伸缩性
  - 故障处理
    - 故障检测
    - 掩盖故障
    - 容错
    - 故障恢复
  - 并发性：服务和应用能并发地处理多个客户请求
  - 透明性：屏蔽分布式系统的组件分离性，使系统被认为是一个整体，而不是独立组件的集合
    - **8种透明性**：
      - 访问透明性
      - 位置透明性
      - 并发透明性
      - 复制透明性
      - 故障透明性
      - 移动透明性
      - 性能透明性
      - 伸缩透明性
  - 服务质量：可靠性、安全性、性能
- 典型分布式系统：分布式文件系统、分布式数据库、分布式缓存系统、分布式协调服务系统、分布式计算框架等
- **分布式文件系统**：
  - GFS：Google File System，基于Linux，提供总体性能较高的服务，未开源
  - HDFS：Hadoop File System，主从结构，NameNode（1，管理文件系统的命名空间和客户端对文件的访问操作） + DataNode（n，管理存储的数据）
  - TFS：Taobao File System，淘宝文件系统，海量小文件
  - MooseFS：管理服务器、数据服务器、元数据备份服务器、客户机
- **分布式缓存**：
  - Memcache：哈希表，物理内存，能存储各种格式的数据（图像、视频、文件以及检索结果等），每个进程最大2G，不支持持久化，LRU淘汰（默认永久，不定期检查，在获取时检查是否过期），分布式由客户端实现（一致性哈希）
  - Redis：多种数据结构（表字符串、哈希、列表、集合和有序集合），快速、可扩展、高效和可靠，可处理高并发用户请求，持久化，高可用架构，主从复制，支持自定义虚拟内存，支持分布式分片集群（一致性哈希），单线程读写性能极高。多线程读写比Memcache慢，持久化为异步写盘，断电可丢失少量数据
  - Tair：分布式key/value结构数据存储系统，支持持久化和非持久化两种使用方式，Config Server + Data Server，桶分片

# 4.9 构件与中间件
- **构件（组件）**：一个自包容、可复用的程序集
- **构件模型**：定义了构件实现、文档化以及开发的标准
  - 接口、使用信息、部署、构件服务
  - 主流模型：Web Services、EJB、.NET
- **构件组装**：
  - 顺序组装、层次组装、叠加组装
  - 接口不兼容问题：参数不兼容、操作不兼容、操作不完备 → 适配器
- **中间件**：为应用提供通用服务和功能的软件。提供了应用系统基本的运行环境，为应用系统提供了更多的高级服务功能
  - **分类**：
    - 通信处理中间件（消息中间件）
    - 事务处理中间件（交易中间件）
    - 数据存取管理中间件
    - Web服务器中间件
    - 安全中间件
    - 跨平台和架构中间件（CORBA、JavaBeans、COM+）
    - 专用平台中间件
    - 网络中间件
  - **中间件与构件的关系**：中间件提供构件运行环境，促进构件化。

# 4.11 云计算
- **云计算**：通过计算机网络形成的计算能力极强的系统，可存储、集合相关资源，并可按需配置，向用户提供个性化服务
- **服务方式**：
  - IaaS（基础设施即服务）：服务提供商将多台服务器组成的“云端”基础设施作为计量服务提供给客户。服务提供商将内存、I/O设备、存储和计算机能力整合为一个虚拟的资源池，为客户提供所需的存储资源、虚拟化服务器等服务
  - PaaS（平台即服务）：服务提供商将分布式开发环境与平台作为一种服务来提供。分布式平台服务，厂商提供开发环境、服务器平台、硬件资源等服务给客户，客户在服务提供商平台的基础上定制开发自己的应用程序，并通过其服务器和互联网传递给其他客户。云计算模式中最为关键的一层，PaaS通常以特定的互联网资源为中心，采用开放平台的形式，对外提供基于Web的API服务
  - SaaS（软件即服务）：服务提供商将应用软件统一部署在云计算平台上，客户根据需要通过互联网向服务提供商订购应用软件服务，服务提供商根据客户所订购的软件的数量、时间的长度等因素收费，并且通过标准浏览器向客户提供应用服务
- **特点**:
  - 灵活性：SaaS < PaaS < IaaS
  - 方便性：IaaS < PaaS < SaaS
- **部署模式**：
  - 公有云
  - 社区云
  - 私有云
  - 混合云
