# 22.1 写作注意事项
- **加强学习**：
  - 经验丰富者：归纳、剖析项目经验
  - 经验欠缺者：阅读文档、临摹
  - 学生：大量阅读范文，强化练习
- **平时积累**：盘点项目、参考案例、多交流
- **提高打字速度**：机考环境，熟悉输入法、快捷键
- **以不变应万变**：准备代表性项目，灵活应对题目

# 22.2 如何解答试题
- **时间分配**：选择3分钟、构思12分钟、摘要15分钟、正文80分钟、检查10分钟
- **试题选择**：选最熟悉、最有把握的
- **论文构思**：论点、项目结合、摘要内容、章节划分
- **写摘要**：简明扼要，阐述论点
- **正文撰写**：按构思追忆素材，控制篇幅
- **检查修改**：遗漏、错别字
- **论文解答实例**：例题分析、答案结构示例（表）

# 22.3 论文写作方法
- **如何写好摘要**：
  - 四种写法（项目+主题+技术+角色）
  - 字数300~400，包含实质性内容
- **如何写好正文**：
  - 以自我为中心（写明自己的贡献）
  - 站在系统分析师角度（全局考虑）
  - 忠实于论点（不跑题）
  - 条理清晰，开门见山
  - 标新立异，有主见
  - 首尾一致
- **摘要和正文的关系**：
  - 摘要不是正文部分，而是抽象
  - 可先写正文后写摘要，或先写摘要后写正文

# 22.4 常见问题及解决办法
- 走题、字数不够/偏多、摘要归纳欠妥、文章深度不够、缺少特色、口语化、文字表达能力
