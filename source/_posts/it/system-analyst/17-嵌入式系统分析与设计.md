# 17.1 嵌入式系统概述
- `嵌入式系统`(Embedded system)是以应用为中心，以计算机为基础，可以剪裁软硬件的专用计算机系统
  - 适用于系统对功能、可靠性、成本、体积、功耗有严格要求的情况
- **特点**：专用性强、实时性强、软硬件依赖、专用处理器、多种技术结合、系统透明、资源受限
  - 专用性强：针对具体应用专业设计的系统，个性化强，软硬件紧密结合
  - 实时性强：外来事件要求在限定时间内及时做出响应，具有实时性
  - 软硬件依赖性强：专用性决定了其软硬件的互相依赖性很强，软硬件协同设计
  - 专用处理器：一般是为嵌入式特定目的而专门设计的嵌入式处理器
  - 多种技术结合：系统通常是计算机技术、半导体技术、电子技术、机械技术与各行业的具体应用相结合的产物
  - 系统透明：与通用计算机系统形态上差别大，甚至没有输出装置，用户无感
  - 资源受限：系统为了达到结构紧凑、高可靠性和低成本的目的，其存储容量、I/O 设备的数量和处理器的处理能力都比较有限
- **组成**：
  - 传感器、处理单元(PE)、作动器、输入接口、输出接口
    - 传感器：将物理参数转换为待处理的电信号
    - 处理单元：负责接受并处理数据
    - 作动器：负责接收电信号，执行物理动作
    - 输入接口：将电信号转换成计算机能处理的二进制数据
    - 输出接口：将二进制数据转换成电信号
  - 体系架构：硬件层、中间层(HAL/BSP)、系统软件层、功能层(应用)
    - 硬件层：嵌入式微处理器、存储器(SDRAM、ROM、Flash等)、通用设备接口和 I/O接口(A/D、D/A、I/O等)
    - 中间层：又称硬件抽象层(Hardware Abstract Layer, HAL)或板级支持包(Board Support Package, BSP)，将上层软件和底层硬件分离开，为上层提供接口
    - 系统软件层：实时多任务操作系统(Real-Time Operation System,RTOS)、文件系统、图形用户接口(Graphic User Interface,GUI)、嵌入式数据库、网络系统及通用组件模块组成。RTOS是嵌入式应用软件的基础和开发平台
    - 功能层：主要由实现某种或某几项任务而被开发运行于操作系统上的程序组成
- **基于多核处理器的嵌入式系统**：
  - 六种结构：单核、多处理器、超线程、多核、共享Cache多核、超线程多核
  - 单核：单芯片单处理器。不支持应用软件并行，任何时间点CPU只能执行一个指令流
  - 多处理器：支持真正意义上的并行，多个线程或进程能够在多个处理器上同时执行
  - 超线程：物理层一个，软件层多个逻辑处理器，支持操作系统和应用程序将多个线程调度到多个逻辑处理器上，就像多处理器系统一样
  - 多核：单芯片多执行核，每个执行核都是独立处理器，有自己的执行集合以及体系结构资源
  - 共享Cache多核：构。与多核结构的工作方式一致，主要差别在于设计时将这些“执行核”设计成可共享片上的Cache
  - 超线程多核：主要是将多核结构与超线程结构相结合
- **安全攸关系统**：
  - 安全攸关系统也称为安全关键系统，是指其不正确的功能或者失效会导致人员伤亡、财产损失等严重后果的计算机系统，是一种安全性、可靠性要求很高的嵌入式系统
  - 安全性(Safety)：指单位时间内损失的期望值小于规定的概率
  - 可靠性设计模式：
    - TMR(Triple Modular Redundancy Pattern, 三模块冗余)
    - 监控-启动器(Monitor Initiator Pattern, 进入安全状态)

# 17.2 嵌入式数据库系统
- 嵌入式数据库管理系统(Embedded Database Management System, EDBMS)是指嵌入式系统中用于存储、管理数据的数据库管理系统。对资源要求相对有限，与操作系统及其应用软件集成在一起
- **特点**：嵌入性、实时性、移动性、伸缩性
- **分类**：
  - 嵌入式内存数据库(FineDB In-Memory、eXtremeDB)
  - 嵌入式实时数据库(eXtremeDB、Empress)
  - 移动数据库(SQL Anywhere、Oracle Lite、SQLite、小金灵)
- **使用环境特点**：设备移动性、网络频繁断接、网络条件多样化、通信能力不对称
- **系统组成**：
  - 嵌入式数据库
  - 同步服务器
  - 数据库服务器
  - 连接网络
- **“小金灵”移动数据库管理系统**：KingBase Lite + SyncServer

# 17.3 嵌入式操作系统
- **分类**：硬实时(VxWorks)、软实时(Android、iOS)
- **特点**：微型化、代码质量高、专业化、实时性强、可裁减和可配置
- **结构**：
  - 硬件抽象层(系统结构抽象、处理器变种抽象、平台抽象)
  - 内核：任务管理、存储管理、任务间通信、异常和中断管理
  - 宏内核 vs 微内核
- **实时性能指标**：
  - 任务切换时间
  - 中断延迟、中断响应、中断恢复
  - 系统响应时间
- **多任务调度算法**：
  - 分类(软/硬时限、周期/非周期、可抢占/非抢占、静态/动态、单/多处理器、在线/离线)
  - **RMS(单调速率调度)**：优先级与周期成反比，可调度性条件 \(\sum E_i/T_i \le n(2^{1/n}-1)\)
  - **时间轮转调度**：时间片，优先级+轮转
  - **EDF(截止时间优先)**：动态优先级，可调度性条件 \(\sum E_i/T_i \le 1\)
  - **分区调度(APPS)**：主时间框架，分区窗口，时间片轮转
- **优先级反转**：
  - 定义：低优先级任务占用资源，高优先级等待
  - 无界反转：中优先级抢占
  - 有界反转：资源占用时间已知
  - 解决方案：
    - 优先级继承协议：低优先级任务继承高优先级
    - 优先级天花板协议：系统级天花板，避免死锁

# 17.4 嵌入式系统开发
- **开发平台**：
  - 交叉开发环境(宿主机 + 目标机)
  - 交叉编译环境(编译器、链接器、目标文件格式COFF/ELF)
- **开发流程**：
  - 半螺旋模型(ROOPES)：需求分析、系统工程一次，螺旋迭代创建原型
  - 分析与设计方法：
    - 改进的结构化方法(RTSAD、DARTS、CODARTS)
    - 面向对象方法(ROOM、OOTRTS)
    - 基于构件的方法
    - 基于模型的开发(MBD)(UML、SysML、AADL)
- **软硬件协同设计**：
  - 方法：软硬件统一描述、综合、验证
  - 工具：协同合成(POLIS、COSYMA)、协同模拟(PTOLEMY)
- **系统分析与设计**：
  - 需求分析(系统规格说明书)
  - 系统架构设计(实时性、设计模型、EOS选择)
  - 硬件子系统设计(处理器选择、接口)
  - 软件子系统设计(任务划分、可调度性分析、优先级分配)
  - EOS选择(功能、工具、可移植性、内存需求、附加软件包、实时性、灵活性)
  - 编程语言选择(通用性、可移植性、执行效率、可维护性)
- **低功耗设计**：
  - 硬件：板级电路、低功耗处理器、总线、接口、分区分时供电
  - 软件：编译优化、软硬件协同、算法优化

# 17.5 嵌入式系统验证
- **验证和确认**：
  - 验证：是否满足规定要求(正确地做事)
  - 确认：是否满足预期用途(做正确的事)
  - 举例(GB/T39086—2020过温保护测试要求)
- **软件测试**：
  - 双V型方法：单元测试、集成测试、系统测试、验收测试
  - 基于需求的测试(DO-178B/C)：
    - 软硬件集成测试、软件集成测试、低级需求测试
    - 覆盖率分析：语句覆盖、判定覆盖、MC/DC(改进条件/判定覆盖)
    - A级软件要求100% MC/DC
  - 形式化方法：
    - 分类：面向对象(Z、VDM)、面向属性(OBJ3)、基于并发(CCS、CSP)、基于实时(TRIO)
    - 模型描述、性质描述
- **基于系统的构件测试方法**：
  - 构件测试目的：质量、标准、选择、复用、修改
  - 不确定问题：异构性、源代码不可得、演化性、不确定、充分性准则难定
  - 测试过程：单元测试、集成测试、系统测试、回归测试
