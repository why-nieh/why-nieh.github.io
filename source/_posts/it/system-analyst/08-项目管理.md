# 8.1 项目管理概述
- **定义**：在有限资源约束下，运用专门知识、技能、工具和方法，实现或超过设定的需求和期望的过程。
- **软件项目管理四要素**：人员、产品、过程、项目。

# 8.2 范围管理
- **范围计划编制**：范围管理计划（如何定义、核实、控制范围）
- **创建工作分解结构（WBS）**：
    - WBS是面向可交付物的层次分解
    - 最底层：工作包（可分配、可管理）
    - 创建方式：白板、草图、项目管理软件
- **范围确认**：
    - 检查可交付成果是否满足要求
    - 检查内容：可核实、有里程碑、有质量标准、审核承诺、覆盖所有活动、风险
- **范围控制**：
    - 变更原因：环境变化、计划不周、新技术、组织变化、用户要求变化
    - 控制原则：决不让步，除非交换；所有变更记载、评估影响

# 8.3 进度管理
- **活动定义**：确定完成项目可交付成果所需的具体活动
- **活动排序**：
    - **前导图法（PDM）**：
        - 节点表示活动，箭线表示依赖关系
        - 四种依赖：FS、FF、SS、SF
    - **箭线图法（ADM）**：
        - 箭线表示活动，节点表示事件
        - 虚活动表示逻辑关系
    - **依赖关系分类**：
        - 强制性依赖（硬逻辑）
        - 可自由处理的依赖（软逻辑）
        - 外部依赖（项目与非项目之间）
- **活动资源估算**：
    - 方法：专家判断、替换方案、公开估算数据、估算软件、自下而上
- **活动历时估算**：
    - **软件项目工作量估算**：
        - 代码行（LOC）估算
        - 功能点估算（输入、输出、查询、文件、接口，乘以复杂度调整因子）
    - **德尔菲法**：
        - 专家匿名估算，求期望 \(E_i = (a_i+4m_i+b_i)/6\)，取中值
        - 多轮反馈
    - **类比估算法**：
        - 与历史项目比较，计算等价代码行
    - **COCOMO模型**：
        - 基本：\(M = a\times (kLOC)^b\)
        - 中间：\(M = a\times (kLOC)^b \times EAF\)（调整因子）
        - 详细：分阶段分层调整
- **进度控制**：
    - 分析进度偏差：
        - 若偏差在关键活动上，必须调整
        - 若偏差大于总时差，影响总工期
        - 若偏差大于自由时差，影响后续活动
    - 调整计划：
        - 关键活动提前/落后：缩短后续关键活动、重新计算
        - 非关键活动调整：在时差范围内延长、缩短、调整开始完成时间
        - 增减工作项目
        - 资源调整（资源优化）

# 8.4 预算与成本管理
- **成本估算**：
    - 自顶向下（类比）
    - 自底向上（汇总）
    - 差别估算法（区分类似与不同部分）
- **成本预算**：
    - 分摊总成本到WBS工作包
    - 分配工作包成本到活动
    - 编制时间计划成本
    - 考虑：直接成本、间接成本、沉没成本、学习曲线、时限、质量要求、管理储备
    - 零基准预算：不以过去为基准
- **成本控制**：
    - 挣值分析：
        - 计划值（PV）
        - 挣值（EV）
        - 实际成本（AC）
        - 成本偏差（CV=EV-AC）
        - 进度偏差（SV=EV-PV）
        - 成本绩效指数（CPI=EV/AC）
        - 进度绩效指数（SPI=EV/PV）

# 8.5 配置管理
- **配置管理内容**：
    - 配置标识、配置控制、配置状态说明、配置审核、版本控制、发行管理
- **配置管理计划**：12部分（引言、引用文件、管理、活动、工具、供货单位控制、记录、配置项和基线、备份、日程表、注解、附录）
- **配置标识**：
    - 配置项：需要受控的文档、代码等
    - 基线：功能基线、分配基线、产品基线
    - 配置库：开发库、受控库、产品库
- **配置管理过程**：
    - 编制计划
    - 配置标识
    - 变更管理和配置控制
    - 配置状态说明
    - 配置审核
    - 版本控制和发行管理

# 8.6 质量管理
- **软件质量模型**：
    - **GB/T25000.10—2016**：
        - 使用质量模型：有效性、效率、满意度、抗风险、周境覆盖
        - 产品质量模型：功能性、性能效率、兼容性、易用性、可靠性、信息安全性、维护性、可移植性
    - **McCall质量模型**：
        - 软件运行：正确性、可靠性、效率、完整性、可用性
        - 软件修改：可维护性、可测试性、灵活性
        - 软件转移：可移植性、可重用性、互操作性
- **质量度量**：
    - 预测型、验收型
    - 尺度度量、二元度量
- **质量保证**：SQA（软件质量保证），有计划、有组织的活动
- **质量控制**：实时监控项目结果，与标准比较，消除不合格原因
- **质量保证与质量控制关系**：QA是过程保证，QC是产品检查；QA指导QC，QC结果反馈给QA

# 8.7 人力资源管理
- **人力资源计划**：
    - 组织结构图
    - 责任分配矩阵（RAM、RACI）
- **组建项目团队**：获取人力资源，动态调整
- **项目团队建设**：
    - 团队发展阶段：形成、震荡、正规、表现
    - 建设措施：培训、集中办公、团队活动、奖励
- **管理项目团队**：
    - 资源负荷（资源需求超过可用资源）
    - 资源平衡（调整进度，解决超负荷）
- **沟通管理**：
    - 正式沟通、非正式沟通
    - 会议、报告

# 8.8 风险管理
- **风险定义**：不确定性，有积极和消极两面
- **风险成本**：有形成本、无形成本、预防费用
- **风险管理过程**：
    1. 风险管理计划编制（角色、预算、类别、概率影响定义）
    2. 风险识别（收集资料、估计形势、识别潜在风险）
    3. 风险定性分析（可能性与影响、优先级、类型）
    4. 风险定量分析（灵敏度、期望货币价值、决策树、蒙特卡罗模拟）
    5. 风险应对计划编制（防范策略：避免、转移、减轻；响应策略：开拓、分享、提高）
    6. 风险监控（跟踪、监测残余风险、识别新风险）

# 8.9 信息（文档）管理
- **软件文档标准**：GB/T8567—2006（25种文档格式）
- **数据需求说明**：逻辑描述、数据采集
- **技术报告**：内容结构（引言、技术问题、系统功能、技术问题与解决方法、现状与问题、总结）
- **项目开发总结报告**：实际开发结果、评价、缺陷处理、经验教训
