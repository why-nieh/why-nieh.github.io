# 14.1 软件实现概述
- **程序设计方法**：
  - 结构化（三种基本控制结构）
  - 面向对象（类、对象、继承、多态）
  - 可视化（事件驱动，表单、部件、属性、事件、方法）
- **程序设计语言选择**：考虑应用领域、用户要求、可用的编译器、软件工具、程序员知识、可移植性
- **程序设计风格**：
  - 可理解性：有意义标识符、注释、清晰结构
  - 可复用性：高内聚、接口分离
  - 可扩展性：封装、多态
  - 健壮性：预防错误、参数检查
- **编码规范**：命名、注释、格式、错误处理、通用开发模式
- **代码生成**：模板 + 数据 → 代码文件
- **代码重用**：函数、对象、模块、类、算法

# 14.2 软件测试概述
- **测试概念**：寻找缺陷，提高质量
- **测试目的**：
  - 用户：发现错误，判断是否可接受
  - 开发者：证明软件无错
  - 测试者：发现错误，预测可靠性
- **测试原则**（10条）：
  1. 测试用例必须包含预期输出
  2. 程序员避免测试自己的程序
  3. 编写软件的组织不应当测试自己编写的软件
  4. 仔细检查每个测试结果
  5. 测试用例应包括有效和无效输入
  6. 避免重新设计测试用例（保留用例）
  7. 计划测试时不应默认不会发现错误
  8. 测试不要穷举
  9. 测试尽早执行
  10. 测试应追溯到需求

# 14.3 软件测试方法
- **按程序是否可见**：
  - 黑盒测试：功能测试，等价类、边界值、因果图、判定表、正交试验、功能图、场景法
  - 白盒测试：结构测试，逻辑覆盖（语句、判定、条件、判定/条件、条件组合、MC/DC）、基本路径、循环覆盖
  - 灰盒测试：介于两者之间
- **按是否需要执行**：
  - 静态测试：代码检查、静态结构分析、质量度量
  - 动态测试：运行程序，输入输出比较

# 14.4 软件测试类型
- **按测试对象**：
  - 功能测试
  - 性能测试（负载、压力、并发、容量、可靠性）
  - 安全测试（渗透、SQL注入、XSS等）
  - 兼容性测试（操作系统、浏览器、硬件）
  - 界面测试（UI）
  - 易用性测试
  - 稳定性测试
- **按测试阶段**：
  - 单元测试（最小模块，白盒为主）
  - 集成测试（模块间接口，灰盒）
  - 系统测试（整个系统，黑盒）
  - 验收测试（α测试、β测试）
- **按被测软件**：
  - App测试、Web测试、物联网测试、车联网测试、大数据测试、AI测试、小程序测试
- **其他分类**：
  - 回归测试（全部、基于风险、基于操作剖面、再测试修改部分）
  - 冒烟测试（预测试，保证版本稳定）

# 14.5 软件测试的组织
- **测试过程**：需求分析、计划、设计文档、搭建环境、执行、缺陷管理
- **测试管理**：
  - 配置管理（控制变更，建立基线）
  - 过程管理（启动、计划、设计、执行、结果分析）
  - 缺陷管理（优先级、状态、跟踪）
  - 风险管理（识别、评估、应对、上报、规避、转移、减轻、接受）

# 14.6 软件部署
- **部署目标**：保障运行、简化操作、满足需求
- **部署步骤**：
  1. 环境准备（操作系统、数据库、网络）
  2. 软件安装
  3. 数据库和文件配置
  4. 测试和验证
