# 20.1 微服务系统概述
- **简介**：将大型应用拆分为小型、自治服务，独立部署、运行
- **优势**：独立性和自治性、弹性和可伸缩性、技术多样性、专用性、可组合性、容错性
- **与单体式系统比较**（对比表）
- **实现方式及平台**：
  - 容器化（Docker、Kubernetes）
  - RESTful API
  - 消息队列（Kafka）
  - 服务注册与发现（Eureka、Consul）
  - 服务网关（Zuul、Nginx）
  - 分布式数据库
  - 平台对比：Spring Cloud vs Kubernetes（表）
- **微服务系统特征**：
  - 服务自治性
  - 单一职责
  - 松耦合
  - 分布式部署
  - 技术异构性
  - 弹性和可伸缩性
  - 独立演化和部署

# 20.2 微服务系统架构
- **架构原则**：
  - 单一职责
  - 隔离性（独立进程、数据库）
  - 自治性（独立生命周期）
  - 弹性（自动伸缩）
  - 可观察性（日志、指标、追踪）
  - 可测试性
  - 可维护性
  - 安全性
  - 智能端点与简单消息传递
- **架构模式**：
  - **聚合器微服务**：聚合器收集多个服务数据，统一返回
  - **代理微服务**：代理网关，转发请求，负载均衡
  - **链式微服务**：服务链式调用，同步阻塞
  - **分支微服务**：根据条件选择不同分支
  - **数据共享微服务**：多个服务共享数据库或数据代理
  - **异步消息微服务**：基于消息队列，事件驱动

# 20.3 微服务系统开发
- **容器化和自动化部署**：
  - Docker（镜像、容器、Dockerfile）
  - Kubernetes（Pod、Service、Volume、Namespace）
  - Jenkins（CI/CD、插件）
- **服务注册和发现**：
  - Consul（服务注册、健康检查、KV）
  - ZooKeeper（Znode、ZAB协议）
  - Eureka（Eureka Server、Eureka Client）
  - Etcd（分布式键值存储，Raft协议）
- **服务通信**：
  - HTTP/REST
  - gRPC（四种通信方式：Unary、Server Streaming、Client Streaming、Bidirectional Streaming）
  - Thrift（IDL、跨语言）
  - Kafka（发布订阅，分区、副本）
- **安全与权限管理**：
  - OAuth2（四种授权模式：授权码、密码、客户端、隐式）
  - JWT（Header、Payload、Signature）
  - TLS/SSL（加密、身份验证）
  - API网关（路由、限流、鉴权）
- **运维监控**：
  - Prometheus（Pull模型、PromQL、Alertmanager）
  - Grafana（可视化、报警）
  - ELK Stack（Elasticsearch、Logstash、Kibana）
  - Zipkin（分布式追踪，Span、Trace）

# 20.4 微服务系统测试
- **测试特点**：分布式、服务自治、异步通信、数据一致性、故障恢复
- **测试过程**：计划、环境搭建、用例设计、执行、结果分析
- **功能测试**：
  - 正常功能测试
  - 异常功能测试
  - 数据一致性测试
  - 异步通信和消息传递测试
  - 接口和集成测试
  - 用户界面测试
- **性能测试**：
  - 负载测试
  - 压力测试
  - 并发性测试
  - 延迟测试
  - 吞吐量测试
  - 性能指标监测
  - 可扩展性测试
- **容错性测试**：
  - 异常输入测试
  - 服务宕机测试
  - 故障注入测试
  - 异常负载测试
  - 回滚和恢复测试
- **安全性测试**：
  - 身份认证与授权
  - 输入验证与过滤
  - 数据保护
  - 安全配置
  - 漏洞扫描/渗透测试
  - 日志和监控
  - 应急响应
